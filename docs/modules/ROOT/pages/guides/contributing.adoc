= Contributing to the Documentation

This guide explains how to author and maintain the AsciiDoc documentation
that powers the https://floatingman-ltd.github.io/arcane-centaur[Arcane Centaur docs site].

== Authoring Workflow

The source of truth for most documentation is the Markdown files in `docs/`
and the root `readme.md`. A GitHub Actions pipeline converts these to AsciiDoc
automatically on every push to `main`.

=== Editing an Existing Page

**For a Markdown-source page** (most pages): edit the `.md` file under `docs/`.
The pipeline will regenerate the corresponding `.adoc` on the next push and
redeploy the site.

**For a manually-graduated page** (no sentinel): edit the `.adoc` file directly
under `docs/modules/ROOT/pages/`. The pipeline will not overwrite it.

=== Adding a New Page

. Create a new `.md` file under `docs/guides/`, `docs/cheatsheets/`, or
  `docs/learning/`.
. The pipeline will generate the `.adoc` automatically on the next push.
. **Update `docs/modules/ROOT/nav.adoc`** to add the new page to the sidebar.
  Without this step the page builds but is unreachable from the navigation.

== Sentinel Header Convention

Auto-generated `.adoc` files start with:

[source,asciidoc]
----
// :auto-generated: true
// :source: docs/guides/example.md
// Remove this header to take manual ownership of this file
----

The pipeline uses this header to decide whether to regenerate the file:

* **Sentinel present** → the file is overwritten from its `.md` source on every push.
* **Sentinel absent** → the file is skipped by the pipeline (manually owned).

=== Graduating a File to Manual AsciiDoc

. Open the `.adoc` under `docs/modules/ROOT/pages/`.
. Remove the three `// :auto-generated:` / `// :source:` / `// Remove this header` lines.
. Commit. The pipeline will no longer touch this file.
. You may now enrich the file with native AsciiDoc features (`NOTE:`, `include::`,
  `[cols=]` table attributes, etc.) that would be lost on regeneration.

NOTE: After graduating a file, changes to its `.md` source will no longer propagate
to the site. Update the `.adoc` directly going forward.

== Local Build

Preview the full site locally before pushing:

[source,sh]
----
./docker/antora/run.sh antora-playbook.yml
# Then open build/site/index.html in a browser
----

The first run pulls the `antora/antora` Docker image (~150 MB). Subsequent runs
use the cache.

== nav.adoc Maintenance

`docs/modules/ROOT/nav.adoc` drives the sidebar navigation. It is authored
manually and must be updated whenever a page is added or removed.

To add a page, insert an `xref:` entry in the appropriate section:

[source,asciidoc]
----
.Guides
* xref:guides/my-new-guide.adoc[My New Guide]
----

Page paths in `nav.adoc` are relative to `docs/modules/ROOT/pages/`.
