== LSP

These keymaps are active in any buffer where an LSP server has attached
(F#, C#, Markdown, Haskell, Janet, Lua).

[cols=",,",options="header",]
|===
|Keys |Mode |Action
|`gd` |Normal |Go to definition
|`K` |Normal |Hover documentation
|`gr` |Normal |Find all references
|`++<++leader++>++rn` |Normal |Rename symbol
|`++<++leader++>++ca` |Normal |Code action
|`++<++leader++>++e` |Normal |Show diagnostic float
|`++[++d` |Normal |Jump to previous diagnostic
|`++]++d` |Normal |Jump to next diagnostic
|===

=== Configured LSP Servers

[width="100%",cols="34%,33%,33%",options="header",]
|===
|Server |Language |Install
|`fsautocomplete` |F# |`dotnet tool install -g fsautocomplete`

|`janet_lsp` |Janet |`jpm install janet-lsp`
|`haskell-language-server` |Haskell |`ghcup install hls` (managed by haskell-tools.nvim)
|`lua_ls` |Lua |See xref:languages/lua.adoc[Lua guide]
|===

== Diagnostics Panel (trouble.nvim)

https://github.com/folke/trouble.nvim[trouble.nvim] v3 provides a structured
panel for diagnostics, references, quickfix, and todos.

[cols=",,",options="header",]
|===
|Keys |Mode |Action
|`++<++leader++>++xx` |Normal |Project diagnostics panel
|`++<++leader++>++xX` |Normal |Buffer diagnostics panel
|`++<++leader++>++xs` |Normal |Symbols panel
|`++<++leader++>++xr` |Normal |LSP references / definitions panel
|`++<++leader++>++xl` |Normal |Location list panel
|`++<++leader++>++xq` |Normal |Quickfix list panel
|`++<++leader++>++xt` |Normal |Todo list (via Trouble)
|`++<++leader++>++xT` |Normal |Todo list (via fzf-lua)
|===

=== Using the panel

Every `++<++leader++>++x*` map is a *toggle*: press it once to open the panel,
again to close it. Inside the panel, `j`/`k` move between entries, `Enter` jumps
to the location under the cursor (the panel stays open), `q` closes the panel,
and `?` lists every in-panel keybinding (preview, fold, filter, delete, …).

Pick the source for the job:

[cols=",",options="header",]
|===
|Panel |What it shows
|Project diagnostics (`++<++leader++>++xx`) |Every LSP warning/error across the whole workspace
|Buffer diagnostics (`++<++leader++>++xX`) |Diagnostics for the current file only
|Symbols (`++<++leader++>++xs`) |Outline of the current file (functions, classes, …); opens to the side and leaves the cursor in your code
|LSP refs/defs (`++<++leader++>++xr`) |References and definitions of the symbol under the cursor, in a right-hand panel
|Location list (`++<++leader++>++xl`) |The current window's location list
|Quickfix (`++<++leader++>++xq`) |The quickfix list — fill it first with `gr`, `:grep`, `:make`, or `:lua vim.diagnostic.setqflist()`
|===

=== Todo comments (todo-comments.nvim)

https://github.com/folke/todo-comments.nvim[todo-comments.nvim] highlights
annotation keywords in your comments — coloured text plus a sign in the gutter —
and can list them across the whole project. A keyword is only recognised when
written with a trailing colon (`TODO:`) inside a comment.

[cols=",",options="header",]
|===
|Keyword |Aliases (same colour)
|`TODO:` |—
|`FIX:` |`FIXME:` `BUG:` `FIXIT:` `ISSUE:`
|`HACK:` |—
|`WARN:` |`WARNING:` `XXX:`
|`PERF:` |`OPTIM:` `PERFORMANCE:` `OPTIMIZE:`
|`NOTE:` |`INFO:`
|`TEST:` |`TESTING:` `PASSED:` `FAILED:`
|===

Two list views scan the whole project for these markers:

* `++<++leader++>++xt` (`:TodoTrouble`) — todos in a Trouble panel, navigated exactly like the panels above.
* `++<++leader++>++xT` (`:TodoFzfLua`) — todos in an fzf-lua fuzzy picker; type to filter, `Enter` to jump.

NOTE: Both views search with *ripgrep*, so `rg` must be on your `PATH`; the
fzf-lua view additionally needs the *fzf* binary. If either is missing the map
reports it (`rg was not found` / `'fzf' not installed`) rather than listing todos.

NOTE: `[d`/`]d` and `<leader>e` (LSP diagnostics float) are unchanged.
`]t`/`[t` are tag navigation, covered in <<tag-navigation>> below.

_Configured in `lua/plugins/trouble.lua` and `lua/plugins/todo-comments.lua`._

== Tag Navigation (ctags)

A `tags` file is an index — generated by
https://github.com/universal-ctags/ctags[universal-ctags] — that maps every
symbol name to the file and line where it is defined. It gives cross-file
"jump to definition" in *any* language with no language server involved, and it
is what makes the `]t`/`[t` maps do anything.

Generate the index at the project root (Neovim finds `./tags` automatically).
The index is a static snapshot, so re-run this after significant edits:

[source,sh]
----
ctags -R          # writes ./tags (gitignored)
----

universal-ctags is a soft dependency — see
xref:getting-started.adoc#system-deps[System Dependencies] to install it.

[cols=",",options="header",]
|===
|Keys |Action
|`Ctrl-]` |Jump to the definition of the tag under the cursor
|`g Ctrl-]` |Same, but list the matches to choose from when there are several (`:tjump`)
|`Ctrl-o` |Jump back to where you came from (the jump list)
|`++]++t` / `++[++t` |Next / previous *match* when a tag has several definitions (vim-unimpaired → `:tnext` / `:tprevious`)
|`++]++T` / `++[++T` |Last / first match (`:tlast` / `:tfirst`)
|`:tag {name}` |Jump to a named tag; `:tag /pat` jumps to the first tag matching a pattern
|`:tselect {name}` |List all matching tags and pick one
|===

NOTE: `]t`/`[t` only move when the current tag has *more than one* definition —
with a single match there is nothing to cycle to (you may see `E428`/`E425` at
the list ends, which is normal). In LSP buffers, `gd` (go to definition) is
usually the better tool and needs no `tags` file.

CAUTION: Vim's built-in `Ctrl-t` (pop the tag stack) is remapped in this
configuration to toggle the file tree (`:NvimTreeToggle`). To step back after a
tag jump use `Ctrl-o` or `:pop` instead.

_`]t`/`[t`/`]T`/`[T` come from `tpope/vim-unimpaired`; `Ctrl-]`, `g Ctrl-]`, and `Ctrl-o` are Vim built-ins._

== Auto-Completion (blink.cmp)

Powered by https://github.com/saghen/blink.cmp[blink.cmp] v1 — a Rust-based
fuzzy completion engine with sub-millisecond latency. Replaces nvim-cmp.

=== Completion Menu

[cols=",,",options="header",]
|===
|Keys |Mode |Action
|`Alt-Space` |Insert |Open completion menu on demand
|`Ctrl-n` |Insert (menu open) |Select next suggestion
|`Ctrl-p` |Insert (menu open) |Select previous suggestion
|`Enter` |Insert |Confirm highlighted suggestion
|`Ctrl-e` |Insert |Dismiss completion menu
|`Ctrl-f` |Insert |Scroll documentation preview down
|`Ctrl-b` |Insert |Scroll documentation preview up
|===

`Enter` only inserts when an entry is actively highlighted — it will not
accidentally confirm the first suggestion (`preselect = false`).

=== Completion Sources

[cols=",",options="header",]
|===
|Source |Provides
|`lsp` |Language server completions (with capabilities advertised to servers)
|`snippets` |Built-in snippet engine (replaces dead LuaSnip config)
|`buffer` |Words from open buffers
|`path` |File-system paths
|`spell` |Dictionary words when `:set spell` is active (min 3 chars); bridged via blink.compat
|`conjure` |Conjure REPL completions (Lisp/Clojure/Scheme/Fennel/Janet buffers); bridged via blink.compat
|===

=== Command-line Completion

[cols=",",options="header",]
|===
|Mode |Sources
|`/` / `?` search |Buffer words
|`:` commands |File paths, then Ex commands
|===

_Configured in `lua/plugins/blink.lua`. LSP capabilities wired in `lua/config/lsp.lua`._

=== Format-on-Save

Format-on-save is enabled automatically for the following filetypes:

[cols=",",options="header",]
|===
|Filetype |Formatter
|Lisp |LSP (if attached)
|Clojure |LSP
|Scheme |LSP
|Fennel |LSP
|Janet |LSP
|F# |LSP (`fsautocomplete`)
|Lua |`stylua`
|===

The formatter runs with a 2-second timeout on save. To disable
format-on-save for a session, use `:ConformDisable`.

_Configured in `lua/plugins/conform.lua`._
