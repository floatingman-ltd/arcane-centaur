=== Layout stability

* `:Bd` ‚Äî layout-preserving buffer delete. Switches the window to
  another listed buffer (or a new empty one) before deleting, so the
  window and the tree's width survive.
* The tree closes automatically when quitting would leave it (and/or
  the terminal panel) as the only remaining window ‚Äî no more full-width
  tree after `:q` on the last editor.

== Terminal

[width="100%",cols="34%,33%,33%",options="header",]
|===
|Keys |Mode |Action
|`++<++leader++>++t` |Normal |Toggle full-width terminal (persistent shell)
|`Esc` |Terminal |Exit terminal insert mode
|===

The terminal always opens as a full-width bottom split regardless of
which window has focus. The shell buffer is reused across open/close
cycles ‚Äî history and running jobs survive window toggles. Press
`++<++C-k++>++` after `Esc` to jump back to the editor window.

== Treesitter Text Objects (nvim-treesitter-textobjects)

Text objects and motions are available in **F#, Haskell, C#, and Lua** buffers.
They are **disabled** in Lisp-family buffers (lisp/clojure/scheme/fennel/janet)
where vim-sexp provides structural `af`/`if` form objects instead.

NOTE: This config tracks nvim-treesitter's maintained `main` branch (not the
frozen `master`), which requires **Neovim ‚â• 0.11** ‚Äî already satisfied by this
config's ‚â• 0.12 baseline.

=== Select objects

Use with any operator (`d`, `c`, `y`, `v`):

[cols=",",options="header",]
|===
|Keys |Object
|`af` / `if` |Outer / inner function
|`ac` / `ic` |Outer / inner class
|`aa` / `ia` |Outer / inner argument (treesitter's `@parameter` capture)
|===

Examples: `daf` deletes a whole function; `vif` selects the function body; `cia`
changes an argument.

=== Move motions

Jump between functions; each jump is added to the jumplist (`Ctrl-o` / `Ctrl-i`
to navigate).

[cols=",",options="header",]
|===
|Keys |Action
|`++]++f` |Next function start
|`++[++f` |Previous function start
|`++]++F` |Next function end
|`++[++F` |Previous function end
|===

NOTE: Class motions (`++]++c` / `++[++c`) are intentionally absent ‚Äî those keys
are reserved for Vim's built-in diff-mode change navigation.

_Plugin: `nvim-treesitter/nvim-treesitter-textobjects`, pinned to its
maintained `main` branch (queries Neovim's core treesitter APIs, so it works
on 0.12 ‚Äî the frozen `master` branch does not). Configured in
`lua/plugins/treesitter.lua`._

== Unimpaired (vim-unimpaired)

vim-unimpaired provides pairs of bracket mappings ‚Äî `++[++` for
"`previous / enable`" and `++]++` for "`next / disable`".

=== Buffer navigation

[cols=",",options="header",]
|===
|Keys |Action
|`++[++b` |Previous buffer
|`++]++b` |Next buffer
|`++[++B` |First buffer
|`++]++B` |Last buffer
|===

=== Quickfix / location list

[cols=",",options="header",]
|===
|Keys |Action
|`++[++q` |Previous quickfix entry
|`++]++q` |Next quickfix entry
|`++[++Q` |First quickfix entry
|`++]++Q` |Last quickfix entry
|`++[++l` |Previous location list entry
|`++]++l` |Next location list entry
|===

The stepping keys only move *through* entries ‚Äî you fill the list first with a
command. Common real-world producers:

[cols="1,2",options="header",]
|===
|Command |What lands in the quickfix list
|`gr` (LSP references) |Every use of the symbol under the cursor. Walk the call
sites with `++]++q` before renaming or deleting it.
|`:lua vim.diagnostic.setqflist()` |All LSP errors/warnings across loaded buffers;
step through them with `++]++q` and fix each in turn.
|`:grep {pat}` / `:vimgrep /{pat}/ **/*.lua` |Project-wide text search. Review hits
with `++]++q`, then apply one edit to every match with `:cdo s/old/new/g \| update`.
|`:make` |Compiler / build errors parsed via `errorformat` ‚Äî jump straight to each
failing line.
|===

TIP: To jump to the next/previous *occurrence of the word under the cursor* ‚Äî a
different job ‚Äî use Vim's built-in `++*++` / `++#++` and `n` / `N` (no list
needed). `++]++d` / `++[++d` step between LSP diagnostics.

=== Argument list

|`++]++a` |Next file in argument list

=== Line operations

[cols=",",options="header",]
|===
|Keys |Action
|`++[<++Space++>++` |Add blank line above cursor
|`++]<++Space++>++` |Add blank line below cursor
|`++[++e` |Exchange current line with line above
|`++]++e` |Exchange current line with line below
|===

=== Option toggles

Prefix with `++[++o` to enable, `++]++o` to disable, `yo` to toggle.

[cols=",",options="header",]
|===
|Keys |Option toggled
|`++[++oh` / `++]++oh` / `yoh` |`hlsearch` (search highlight)
|`++[++oi` / `++]++oi` / `yoi` |`ignorecase`
|`++[++on` / `++]++on` / `yon` |`number` (line numbers)
|`++[++or` / `++]++or` / `yor` |`relativenumber`
|`++[++os` / `++]++os` / `yos` |`spell`
|`++[++ow` / `++]++ow` / `yow` |`wrap`
|`++[++ox` / `++]++ox` / `yox` |`cursorline` {plus} `cursorcolumn`
|===

=== URL / XML encoding

[cols=",",options="header",]
|===
|Keys |Action
|`++[++u++{motion}++` |URL-encode text
|`++]++u++{motion}++` |URL-decode text
|`++[++x++{motion}++` |XML-encode text (`&amp;`, `&lt;`, etc.)
|`++]++x++{motion}++` |XML-decode text
|===

_Plugin loaded via `lua/plugins/init.lua`. No custom configuration ‚Äî
default vim-unimpaired keybindings apply._
