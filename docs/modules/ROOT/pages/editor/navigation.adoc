== IDE Layout

Press `++<++leader++>++L` to assemble the IDE workspace in one keystroke:

....
┌────────────┬───────────────────┐
│  nvim-tree │      editor       │
├────────────┴───────────────────┤
│   terminal (full width, 15)    │
└────────────────────────────────┘
....

The operation is idempotent — invoking it when the layout is already
assembled does nothing, and focus always returns to the editor window.

=== Layout stability

* `:Bd` — layout-preserving buffer delete. Switches the window to
  another listed buffer (or a new empty one) before deleting, so the
  window and the tree's width survive.
* The tree closes automatically when quitting would leave it (and/or
  the terminal panel) as the only remaining window — no more full-width
  tree after `:q` on the last editor.

== Terminal

[width="100%",cols="34%,33%,33%",options="header",]
|===
|Keys |Mode |Action
|`++<++leader++>++t` |Normal |Toggle full-width terminal (persistent shell)
|`Esc` |Terminal |Exit terminal insert mode
|===

The terminal always opens as a full-width bottom split regardless of
which window has focus. The shell buffer is reused across open/close
cycles — history and running jobs survive window toggles. Press
`++<++C-k++>++` after `Esc` to jump back to the editor window.

== Fuzzy Finder (fzf-lua)

fzf-lua uses the default fzf key bindings inside its popup window. Open
the picker with any `:FzfLua ++<++command++>++` call.

[cols=",",options="header",]
|===
|Keys |Action
|`Ctrl-j` / `↓` |Move to next item
|`Ctrl-k` / `↑` |Move to previous item
|`Enter` |Open selected file (default split)
|`Ctrl-v` |Open selected file in vertical split
|`Ctrl-s` / `Ctrl-x` |Open selected file in horizontal split
|`Ctrl-t` |Open selected file in new tab
|`Ctrl-c` / `Esc` |Close the picker
|`Ctrl-a` |Select all items
|`Ctrl-d` |Deselect all items
|===

=== Example Commands

....
:FzfLua files          — fuzzy-find files
:FzfLua live_grep      — live ripgrep across project
:FzfLua buffers        — switch between open buffers
:FzfLua lsp_references — LSP references for symbol under cursor
:FzfLua help_tags      — search Neovim help topics
....

_Plugin configured in `lua/plugins/fzf-lua.lua`._

== Treesitter Text Objects (nvim-treesitter-textobjects)

Text objects and motions are available in **F#, Haskell, C#, and Lua** buffers.
They are **disabled** in Lisp-family buffers (lisp/clojure/scheme/fennel/janet)
where vim-sexp provides structural `af`/`if` form objects instead.

=== Select objects

Use with any operator (`d`, `c`, `y`, `v`):

[cols=",",options="header",]
|===
|Keys |Object
|`af` / `if` |Outer / inner function
|`ac` / `ic` |Outer / inner class
|`aa` / `ia` |Outer / inner argument (parameter)
|===

Examples: `daf` deletes a whole function; `vif` selects the function body; `cia`
changes an argument.

=== Move motions

Jump between functions; each jump is added to the jumplist (`Ctrl-o` / `Ctrl-i`
to navigate).

[cols=",",options="header",]
|===
|Keys |Action
|`++]++f` |Next function start
|`++[++f` |Previous function start
|`++]++F` |Next function end
|`++[++F` |Previous function end
|===

NOTE: Class motions (`++]++c` / `++[++c`) are intentionally absent — those keys
are reserved for Vim's built-in diff-mode change navigation.

_Plugin: `nvim-treesitter/nvim-treesitter-textobjects`, configured in
`lua/plugins/treesitter.lua`._

== Unimpaired (vim-unimpaired)

vim-unimpaired provides pairs of bracket mappings — `++[++` for
"`previous / enable`" and `++]++` for "`next / disable`".

=== Buffer navigation

[cols=",",options="header",]
|===
|Keys |Action
|`++[++b` |Previous buffer
|`++]++b` |Next buffer
|`++[++B` |First buffer
|`++]++B` |Last buffer
|===

=== Quickfix / location list

[cols=",",options="header",]
|===
|Keys |Action
|`++[++q` |Previous quickfix entry
|`++]++q` |Next quickfix entry
|`++[++Q` |First quickfix entry
|`++]++Q` |Last quickfix entry
|`++[++l` |Previous location list entry
|`++]++l` |Next location list entry
|===

=== Argument list

[cols=",",options="header",]
|===
|Keys |Action
|`++[++a` |Previous file in argument list
|`++]++a` |Next file in argument list
|===

=== Line operations

[cols=",",options="header",]
|===
|Keys |Action
|`++[<++Space++>++` |Add blank line above cursor
|`++]<++Space++>++` |Add blank line below cursor
|`++[++e` |Exchange current line with line above
|`++]++e` |Exchange current line with line below
|===

=== Option toggles

Prefix with `++[++o` to enable, `++]++o` to disable, `yo` to toggle.

[cols=",",options="header",]
|===
|Keys |Option toggled
|`++[++oh` / `++]++oh` / `yoh` |`hlsearch` (search highlight)
|`++[++oi` / `++]++oi` / `yoi` |`ignorecase`
|`++[++on` / `++]++on` / `yon` |`number` (line numbers)
|`++[++or` / `++]++or` / `yor` |`relativenumber`
|`++[++os` / `++]++os` / `yos` |`spell`
|`++[++ow` / `++]++ow` / `yow` |`wrap`
|`++[++ox` / `++]++ox` / `yox` |`cursorline` {plus} `cursorcolumn`
|===

=== URL / XML encoding

[cols=",",options="header",]
|===
|Keys |Action
|`++[++u++{motion}++` |URL-encode text
|`++]++u++{motion}++` |URL-decode text
|`++[++x++{motion}++` |XML-encode text (`&amp;`, `&lt;`, etc.)
|`++]++x++{motion}++` |XML-decode text
|===

_Plugin loaded via `lua/plugins/init.lua`. No custom configuration —
default vim-unimpaired keybindings apply._
