== Visual Mode Editing

[cols=",,",options="header",]
|===
|Keys |Mode |Action
|`ggVG` |Normal |Select all lines (go to top → Visual Line → go to bottom)
|`++<++` |Visual |Outdent selection and reselect
|`++>++` |Visual |Indent selection and reselect
|`++<++Tab++>++` |Visual |Indent selection
|`++<++S-Tab++>++` |Visual |Outdent selection
|`++<++S-Down++>++` |Visual |Move selected block down
|`++<++S-Up++>++` |Visual |Move selected block up
|`++<++leader++>++p` |Visual |Paste without overwriting clipboard
|===

== System Clipboard

With `clipboard=unnamedplus` set, the default `y`, `d`, and `p` already
use the system clipboard on most platforms. The shortcuts below use the
explicit `"{plus}` register for unambiguous clipboard access (useful
when you need to be certain you are targeting the system clipboard, not
just the unnamed register).

[cols=",,",options="header",]
|===
|Keys |Mode |Action
|`++<++leader++>++y` |Normal / Visual |Yank to system clipboard
|`++<++leader++>++Y` |Normal |Yank line to system clipboard
|`++<++leader++>++d` |Normal / Visual |Cut (delete) to system clipboard
|`++<++leader++>++p` |Normal |Paste from system clipboard after cursor
|`++<++leader++>++P` |Normal |Paste from system clipboard before cursor
|===

|===
|Environment |Clipboard tool |Install needed?

|GUI Linux (Wayland) |`wl-clipboard` |Yes —
`sudo apt install wl-clipboard`

|WSL |`win32yank.exe` |Yes — install on the *Windows* side
(Scoop/Chocolatey)

|SSH / TTY / console |OSC 52 (built-in) |No — requires Neovim ≥ 0.10 and

|Native Windows |`win32yank.exe` |Usually bundled with Neovim Windows
installer
|===

== Commenting (Neovim native `gc`)

Neovim's built-in `gc` operator is enabled by default (no extra plugin needed).

[width="100%",cols="34%,33%,33%",options="header",]
|===
|Keys |Mode |Action
|`gcc` |Normal |Toggle comment on current line
|`gc` |Visual |Toggle comment on selection
|`gc++<++motion++>++` |Normal |Toggle comment over a motion (e.g. `gcap` = paragraph, `gc2j` = 3 lines)
|===

Respects the correct comment syntax for each filetype automatically
(`--` for Lua, `//` for C#/JavaScript, `;;` for Lisp, `++#++` for Python, etc.).
All ops are dot-repeatable.

== Surround (nvim-surround)

https://github.com/kylechui/nvim-surround[nvim-surround] adds keybindings to
add, change, and delete surrounding characters (quotes, brackets, tags, etc.)
around text objects. Keymaps are identical to the former vim-surround.

=== Change surroundings

[cols=",,",options="header",]
|===
|Keys |Action |Example
|`cs"'` |Change `"` to `'` |`"hello"` → `'hello'`

|`cs'++<++q++>++` |Change `'` to `++<++q++>++` tags |`'hello'` →
`++<++q++>++hello++<++/q++>++`

|`cst"` |Change tag to `"` |`++<++q++>++hello++<++/q++>++` → `"hello"`
|===

=== Delete surroundings

[cols=",,",options="header",]
|===
|Keys |Action |Example
|`ds"` |Delete surrounding `"` |`"hello"` → `hello`

|`dst` |Delete surrounding HTML tag |`++<++p++>++hello++<++/p++>++` →
`hello`
|===

=== Add surroundings

Cursor must be inside (or on) the text to surround. `ys` = "`you surround`".

[width="100%",cols="34%,33%,33%",options="header",]
|===
|Keys |Action |Example
|`ysiw"` |Surround word with `"` |`hello` → `"hello"`

|`ysiw(` |Surround word with `( )` (with spaces) |`hello` → `( hello )`

|`ysiw)` |Surround word with `()` (no spaces) |`hello` → `(hello)`

|`yss)` |Surround entire line with `()` |`hello world` → `(hello world)`

|`ysit++<++em++>++` |Surround inner tag content with `++<++em++>++`
|`++<++p++>++hello++<++/p++>++` →
`++<++p++><++em++>++hello++<++/em++><++/p++>++`
|===

=== Visual mode

Select text in visual mode, then press `S` followed by the surround character.

[cols=",",options="header",]
|===
|Keys |Action
|`S"` |Surround selection with `"`
|`S)` |Surround selection with `()`
|`S++<++p++>++` |Surround selection with `++<++p++><++/p++>++` tags
|===

_Plugin configured in `lua/plugins/nvim-surround.lua`._
