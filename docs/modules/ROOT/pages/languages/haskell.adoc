== Working with Haskell

Haskell support is provided by
https://github.com/mrcjkb/haskell-tools.nvim[haskell-tools.nvim], which
bundles LSP integration, a GHCi REPL, and Hoogle search into a single
plugin.

All Haskell plugins lazy-load only when you open a `.hs` or `.lhs` file.

[[quick-start]]
== Quick Start

[arabic]
. Install GHC and haskell-language-server via `ghcup` (see
xref:#prerequisites[Prerequisites]).
. Open a Haskell source file — plugins load automatically:
+
[source,sh]
----
nvim src/Main.hs
----
. Press `++<++leader++>++rr` to open a GHCi REPL for the current package.
. Evaluate code in the REPL, or press `++<++leader++>++rf` to open a
file-scoped REPL.

'''''

[[lsp]]
== LSP

`haskell-tools` manages the `hls` LSP connection automatically — no
manual `lspconfig` setup is needed. See the
xref:editor/code-intelligence.adoc[Code Intelligence cheatsheet] for
shared LSP keybindings (`gd`, `K`, `gr`, etc.).

'''''

[[repl]]
== REPL (GHCi)

haskell-tools provides a built-in GHCi REPL. Keybindings are defined in
`after/ftplugin/haskell.lua`.

Leader is `Space`; no LocalLeader bindings are used in Haskell buffers.

[cols=",",options="header",]
|===
|Keys |Action
|`++<++leader++>++rr` |Toggle GHCi REPL for the current package
|`++<++leader++>++rf` |Toggle GHCi REPL for the current file
|`++<++leader++>++rq` |Quit the GHCi REPL
|`Space-cl` |Run code lenses (haskell-language-server)
|`Space-hs` |Hoogle search for type under cursor
|`Space-ea` |Evaluate all code snippets in the buffer
|===

'''''

[[configuration]]
== Configuration

Plugin spec lives in `lua/plugins/haskell.lua` and ftplugin overrides
live in `after/ftplugin/haskell.lua`.

'''''

== Debugging

https://github.com/MrcJkb/haskell-tools.nvim[haskell-tools.nvim] auto-discovers
DAP launch configurations once https://github.com/mfussenegger/nvim-dap[nvim-dap]
is installed (added in change 07 for .NET debug support).

Use the standard debug keymaps (`<F5>`, `<F9>`, `<F10>`/`<F11>`/`<F12>`) to start
a session. No additional Haskell-specific debug adapter install is required for
basic GHCi-based debugging; a full `haskell-debug-adapter` setup is optional and
left for a future change.

'''''
