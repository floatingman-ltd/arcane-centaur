This configuration provides full LSP support and format-on-save for Lua
via `lua-language-server` and `stylua`.

[[quick-start]]
== Quick Start

. *Open a Lua file* — LSP attaches automatically:
+
[source,sh]
----
nvim lua/config/lsp.lua
----
. *Use LSP keymaps* (see xref:../editor/code-intelligence.adoc[Code Intelligence cheatsheet] for the full list):
+
[cols=",",options="header",]
|===
|Keys |Action
|`gd` |Go to definition
|`K` |Hover documentation
|`gr` |Find references
|`<leader>rn` |Rename symbol
|`<leader>ca` |Code action
|`<leader>e` |Show diagnostics
|===
. *Format on save* — `stylua` runs automatically when you write a `.lua` file.
  Manual format: `<leader>f` in normal or visual mode.

'''''

[[lsp]]
== LSP

`lua_ls` (lua-language-server) is configured in `lua/config/lsp.lua` and
attaches automatically when a Lua buffer is opened. It provides
completions, diagnostics, go-to-definition, and hover docs.

Verify the server is attached with:

[source,vim]
----
:lua =vim.lsp.get_clients({bufnr=0})
----

'''''

[[formatting]]
== Formatting

`stylua` runs via conform.nvim on every save (2-second timeout). It also
respects a `stylua.toml` or `.stylua.toml` in the project root.

To format manually:

[cols=",",options="header",]
|===
|Keys |Action
|`<leader>f` |Format buffer or visual selection
|===

Indent settings for Lua buffers (set in `after/ftplugin/lua.lua`):
2-space tabs, spaces only.

'''''

=== Installing lua-language-server

NOTE: `lua-language-server` is *not* available via `apt` or snap on Ubuntu 24.04.
Install from the GitHub releases page.

[cols=",",options="header",]
|===
|Platform |Method
|*Ubuntu / Debian* |Binary download — see below
|*Arch Linux* |`sudo pacman -S lua-language-server`
|*macOS (Homebrew)* |`brew install lua-language-server`
|===

==== Ubuntu / Debian — binary install

Download and extract the latest release (check https://github.com/LuaLS/lua-language-server/releases[releases] for the current version):

[source,sh]
----
VERSION=3.18.2
wget https://github.com/LuaLS/lua-language-server/releases/download/${VERSION}/lua-language-server-${VERSION}-linux-x64.tar.gz
mkdir -p ~/.local/opt/lua-language-server
tar -xzf lua-language-server-${VERSION}-linux-x64.tar.gz -C ~/.local/opt/lua-language-server
----

The binary must run from its extracted location (it resolves scripts relative to itself).
Do **not** symlink the binary directly — create a wrapper script in `~/.local/bin` instead
(`~/.local/bin` is already on `$PATH` on Ubuntu 24.04):

[source,sh]
----
cat > ~/.local/bin/lua-language-server << 'EOF'
#!/bin/sh
exec "$HOME/.local/opt/lua-language-server/bin/lua-language-server" "$@"
EOF
chmod +x ~/.local/bin/lua-language-server
----

Verify:

[source,sh]
----
lua-language-server --version
----

=== Installing stylua

NOTE: `stylua` is not available via `apt` on Ubuntu 24.04. Install the binary directly from GitHub releases:

[source,sh]
----
curl -sLo /tmp/stylua.zip https://github.com/JohnnyMorganz/StyLua/releases/latest/download/stylua-linux-x86_64.zip
unzip /tmp/stylua.zip -d ~/.local/bin/
chmod +x ~/.local/bin/stylua
----

Or via package manager on other platforms:

[cols=",",options="header",]
|===
|Platform |Command
|*Arch Linux* |`sudo pacman -S stylua`
|*macOS (Homebrew)* |`brew install stylua`
|===

Verify:

[source,sh]
----
stylua --version
----
