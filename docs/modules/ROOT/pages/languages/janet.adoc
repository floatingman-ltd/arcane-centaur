== Working with Janet

Janet is a lightweight, expressive Lisp dialect designed for scripting,
system automation, and embedding. This configuration provides
REPL-driven development via Conjure, structural editing, and LSP
support.

[width="100%",cols="50%,50%",options="header",]
|===
|Plugin |Role
|https://github.com/Olical/conjure[Conjure] |Connect to a REPL and
evaluate code without leaving the editor

|https://github.com/guns/vim-sexp[vim-sexp] {plus}
https://github.com/tpope/vim-sexp-mappings-for-regular-people[mappings]
|Structural editing — slurp, barf, and move S-expressions

|https://github.com/gpanders/nvim-parinfer[nvim-parinfer] |Keeps
parentheses balanced automatically as you edit indentation

|https://github.com/HiPhish/rainbow-delimiters.nvim[rainbow-delimiters.nvim]
|Color-codes matching delimiters so you can see nesting at a glance
|===

All four plugins lazy-load only when you open a *Janet* file (`.janet`).

https://github.com/stevearc/conform.nvim[conform.nvim] is also loaded
for Janet and provides *format-on-save* as well as a manual
*`++<++leader++>++f`* keybinding (normal and visual mode) to reformat
the current buffer or selection.

[[quick-start]]
== Quick Start

[arabic]
. *Open a Janet source file* — plugins load automatically:
+
[source,sh]
----
nvim hello.janet
----
. *Evaluate code* using Conjure (local leader is `,`):
+
Conjure spawns a Janet REPL process automatically when you first
evaluate code in a `.janet` buffer — no manual server setup is needed.
+
[cols=",",options="header",]
|===
|Keys |Action
|`,ee` |Evaluate the form under the cursor
|`,er` |Evaluate the root (outermost) form
|`,eb` |Evaluate the entire buffer
|`,e!` |Replace the form with its result
|`,lv` |Open the REPL log in a vertical split
|`,ls` |Open the REPL log in a horizontal split
|`,lq` |Close the REPL log window
|===
. *Edit structure* with vim-sexp (normal mode):
+
[cols=",",options="header",]
|===
|Keys |Action
|`++>++)` |*Slurp forward* — pull the next sibling into the form
|`++<++)` |*Barf forward* — push the last element out of the form
|`++<++(` |*Slurp backward* — pull the previous sibling into the form
|`++>++(` |*Barf backward* — push the first element out of the form
|`++<++f` |Move the current form left among its siblings
|`++>++f` |Move the current form right among its siblings
|`++<++e` |Move the current element left
|`++>++e` |Move the current element right
|`cse(` or `cse)` |Surround the element with `()`
|`cse++[++` or `cse++]++` |Surround the element with `++[]++`
|`dsf` |Delete surrounding function call (splice)
|===
. *Parinfer* runs in the background — just adjust indentation and parens
follow. No keys needed.

'''''

[[lsp]]
== LSP

The `janet++_++lsp` server is configured in `lua/config/lsp.lua`.
Install it via jpm:

[source,sh]
----
jpm install janet-lsp
----

Make sure `janet-lsp` is on your `$PATH` (typically `~/.jpm/bin/`). See
the xref:editor/code-intelligence.adoc[Code Intelligence cheatsheet]
for shared LSP keybindings.

'''''

[[typical-workflow]]
== Typical Workflow

....
 +----------------------------------------------+
 |  Neovim                                      |
 |  +-------------------+------------------+    |
 |  |  source.janet     |  Conjure log     |    |
 |  |                   |  (REPL output)   |    |
 |  |  (defn greet [n]  |  => "Hello!"     |    |
 |  |    (string "Hello |                  |    |
 |  |      " n "!"))    |                  |    |
 |  |                   |                  |    |
 |  |  ,ee to eval -----+                  |    |
 |  +-------------------+------------------+    |
 +----------------------------------------------+
....

[arabic]
. Write or edit code in the left pane.
. Press `,ee` to evaluate the expression under the cursor — the result
appears in the Conjure log.
. Use `,lv` to open the log in a vertical split if it isn't visible.
. Use vim-sexp motions (`++>++)`, `++<++(`, etc.) to restructure
S-expressions without counting parentheses.
. Parinfer keeps parens balanced automatically as you change
indentation.
. Rainbow delimiters let you visually match nesting depth.

'''''

[[troubleshooting]]
== Troubleshooting

=== Conjure does not start the REPL

Make sure `janet` is on your `$PATH`:

[source,sh]
----
which janet
----

If Janet is installed but Neovim cannot find it, check your shell `PATH`
and ensure it is inherited by Neovim.

=== LSP features not working

Verify `janet-lsp` is installed and accessible:

[source,sh]
----
which janet-lsp
----

Use `:lua =vim.lsp.get_clients({bufnr=0})` inside Neovim to check whether
the server is attached to the current buffer. To open the LSP log:

[source,vim]
----
:lua vim.cmd('e ' .. vim.lsp.get_log_path())
----

`:checkhealth lsp` gives a broader overview of LSP status.

'''''

[[prerequisites]]
== Prerequisites

=== Installing Janet

[width="100%",cols="50%,50%",options="header",]
|===
|Platform |Command
|*Ubuntu / Debian* |Build from source — see
https://janet-lang.org/[janet-lang.org]

|*Arch Linux* |`sudo pacman -S janet`

|*Windows* |Download from https://janet-lang.org/[janet-lang.org]

|*macOS (Homebrew)* |`brew install janet`
|===

Verify the installation:

[source,sh]
----
janet -v
----

==== Docker option (explored, not implemented)

https://hub.docker.com/r/leafgarland/janet-sdk[leafgarland/janet-sdk] is
the main community image — no official Docker image exists. It ships
Janet, jpm, git, and a C compiler.

Because Conjure spawns the `janet` binary directly (unlike Common Lisp,
which uses a TCP socket), the Docker REPL would require a wrapper script
on `$PATH` named `janet` that delegates to `docker run`:

[source,sh]
----
#!/bin/sh
docker run --rm -i leafgarland/janet-sdk janet "$@"
----

The LSP server (`janet-lsp`) is harder to containerise — it is a
persistent stdio process that needs access to project files. The simpler
split is to use Docker for the REPL and install `janet-lsp` locally via
jpm. This combination has not been set up or tested for this config.

=== Installing jpm (Janet Package Manager)

`jpm` is required to install the LSP server and manage Janet projects.
It is typically bundled with Janet installations. If it is not
available, build it from source:

[source,sh]
----
git clone https://github.com/janet-lang/jpm.git
cd jpm
janet bootstrap.janet
----

See https://github.com/janet-lang/jpm[janet-lang/jpm] for full
instructions.
