== REPL — iron.nvim

REPL backend: *csharprepl* for C# buffers · *dotnet fsi* for F# buffers.

[cols=",",options="header",]
|===
|Keys |Action
|`,sl` |Send current line to REPL
|`,sc` |Send motion or visual selection to REPL
|`,sp` |Send current paragraph to REPL
|`,sf` |Send entire file to REPL
|`,s++<++CR++>++` |Send carriage return to REPL
|`,si` |Interrupt REPL
|`,sq` |Quit / exit REPL
|`,cl` |Clear REPL output
|===

== Debugging (nvim-dap + netcoredbg)

Prerequisites: netcoredbg (GitHub-release binary — *not* `dotnet tool`), the `EasyDotnet` server tool
(`dotnet tool install -g EasyDotnet`), and `fzf`. Details in the .NET guide.

[cols=",,",options="header",]
|===
|Keys |Mode |Action
|`<F9>` / `++<++leader++>++bb` |Normal |Toggle breakpoint
|`<F5>` / `++<++leader++>++bc` |Normal |Continue / start
|`<F10>` / `++<++leader++>++bv` |Normal |Step over
|`<F11>` / `++<++leader++>++bi` |Normal |Step into
|`<F12>` / `++<++leader++>++bo` |Normal |Step out
|`<S-F5>` / `++<++leader++>++bt` |Normal |Terminate
|`++<++leader++>++bu` |Normal |Toggle debug UI
|`++<++leader++>++br` |Normal |Open REPL
|===

The `++<++leader++>++b` mirrors exist for terminals that grab the F-keys (e.g. `<F11>` fullscreen).
