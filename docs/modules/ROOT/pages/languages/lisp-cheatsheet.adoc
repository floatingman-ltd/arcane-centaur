== Common Lisp REPL Containers

Compose file: `docker/lisp-swank/docker-compose.yml`

*The :4005 contract:* whoever owns `127.0.0.1:4005` is the active Lisp.
Switch by bringing down the current profile and bringing up the next; reconnect with `,cc`.

[cols="1,3",options="header"]
|===
|Implementation |Start command
|SBCL (default)  |`LISP_DIR=$PWD docker compose -f ~/.config/nvim/docker/lisp-swank/docker-compose.yml up -d --wait`
|CCL  |`LISP_DIR=$PWD docker compose -f ~/.config/nvim/docker/lisp-swank/docker-compose.yml --profile ccl  up -d --wait`
|ECL  |`LISP_DIR=$PWD docker compose -f ~/.config/nvim/docker/lisp-swank/docker-compose.yml --profile ecl  up -d --wait`
|ABCL |`LISP_DIR=$PWD docker compose -f ~/.config/nvim/docker/lisp-swank/docker-compose.yml --profile abcl up -d --wait`
|===

[source,sh]
----
# Switch: bring down current, bring up next, reconnect
docker compose -f ~/.config/nvim/docker/lisp-swank/docker-compose.yml down
# ...then the start command for the target impl above
# Then in Neovim: ,cc
----

== Structural Editing — vim-sexp

[width="100%",cols="50%,50%",options="header",]
|===
|Keys |Action
|`++>++)` |*Slurp forward* — extend the closing `)` rightward, pulling
the next sibling _into_ the form

|`++<++)` |*Barf forward* — shrink the closing `)` leftward, pushing the
last element _out of_ the form

|`++<++(` |*Slurp backward* — extend the opening `(` leftward, pulling
the previous sibling _into_ the form

|`++>++(` |*Barf backward* — shrink the opening `(` rightward, pushing
the first element _out of_ the form

|`++>++f` |Move current form right among siblings

|`++<++f` |Move current form left among siblings

|`++>++e` |Move current element right

|`++<++e` |Move current element left

|`cse(` or `cse)` |Surround element with `()`

|`cse++[++` or `cse++]++` |Surround element with `++[]++`

|`dsf` |Delete surrounding function call (splice)
|===

____
See
xref:languages/lisp.adoc#structural-editing[../guides/lisp.md]
for step-by-step examples of each slurp/barf operation.
