The REPL-driven workflow naturally supports a tight test loop: write a function, test it at the REPL,
codify that test, reload, repeat. Common Lisp has no single blessed test framework, but *FiveAM* is
the most widely used and cleanest to work with interactively.

This lesson covers FiveAM basics, integrating tests with ASDF, and the interactive
fix-and-re-run cycle that makes REPL testing effective.

The examples extend the `greeter` system from lesson 09.

----

== 10 — Testing

=== 1. Why Test from the REPL?

In languages with a build step, tests are usually run in a separate process. In Common Lisp, you can:

* Load the test file into the running image with `,eb`
* Fix a failing function interactively
* Re-run the test suite with a single `(fiveam:run!)` call
* Never leave Neovim

The feedback loop looks like: edit form → `,ee` to reload → `(fiveam:run!)` → see green. All from one editor window.

'''

=== 2. FiveAM — Installation and Vocabulary

[source,lisp]
----
;; Load FiveAM from Quicklisp
(ql:quickload :fiveam)
----

FiveAM's core vocabulary:

[cols="2,5",options="header"]
|===
| Form | Purpose

| `(def-suite name)`
| Declare a named test suite

| `(in-suite name)`
| Set the current suite for subsequent `test` definitions

| `(test name &body body)`
| Define a test (a named sequence of assertions)

| `(is expr)`
| Assert that `expr` is true; records failure with the source form if not

| `(is (= a b))`
| Assert equality (generalized with the right comparator)

| `(signals condition-type form)`
| Assert that `form` signals a condition of `condition-type`

| `(finishes form)`
| Assert that `form` completes without signalling any condition

| `(fiveam:run!)`
| Run all defined tests, print a summary, return pass/fail

| `(fiveam:run! 'suite-name)`
| Run only the named suite
|===

'''

=== 3. A First Test Suite

[source,lisp]
----
(ql:quickload :fiveam)
(use-package :fiveam)

;; Declare a top-level suite
(def-suite greeter-suite
  :description "Tests for the greeter package")

;; Enter the suite — subsequent (test ...) forms belong here
(in-suite greeter-suite)

;; The function under test (inline for this scratch session)
(defun greet (name)
  (format nil "Hello, ~A!" name))

;; A simple test
(test greet-with-name
  (is (string= "Hello, Alice!" (greet "Alice")))
  (is (string= "Hello, Bob!"   (greet "Bob")))
  (is (string= "Hello, !"      (greet ""))))   ; empty string edge case

;; Run the suite
(run! 'greeter-suite)
----

*Try it:* evaluate the block above with `,eb` (buffer). The output in the Conjure log should show:

....
Running test suite GREETER-SUITE
 Running test GREET-WITH-NAME .......
 Did 3 checks.
    Pass: 3 (100%)
    Skip: 0 (0%)
    Fail: 0 (0%)
....

'''

=== 4. The is Assertion in Detail

`is` is the primary assertion macro. It wraps an arbitrary form and records success or failure:

[source,lisp]
----
(test is-examples
  ;; Equality
  (is (= 4 (+ 2 2)))
  (is (string= "hello" (string-downcase "HELLO")))
  (is (equal '(1 2 3) (list 1 2 3)))

  ;; Truthiness
  (is (null nil))
  (is (not nil))

  ;; Numeric comparison
  (is (< 1 2 3 4))
  (is (>= 10 10))

  ;; Type check
  (is (typep 42 'integer))
  (is (listp '(a b c))))

(run! 'greeter-suite)
----

When a check fails, FiveAM reports the failing form and the actual value:

[source,lisp]
----
(test will-fail
  (is (= 5 (+ 2 2))))   ; intentional failure

(run! 'greeter-suite)
;; Output includes:
;;  UNEXPECTED-FAILURE in WILL-FAIL:
;;   (= 5 (+ 2 2))
;;   evaluated to NIL
----

'''

=== 5. signals and finishes

`signals` asserts that a form raises a particular condition type:

[source,lisp]
----
(defun safe-divide (a b)
  (when (zerop b)
    (error "Division by zero"))
  (/ a b))

(test safe-divide-tests
  (is (= 5 (safe-divide 10 2)))
  (signals error (safe-divide 10 0)))

(run! 'greeter-suite)
----

`finishes` asserts that a form completes without any unhandled condition:

[source,lisp]
----
(test finishes-example
  (finishes (greet "World"))    ; must not signal
  (finishes (format nil "~A" 42)))

(run! 'greeter-suite)
----

'''

=== 6. A Complete Test Suite for the greeter

Create the test files in the `greeter/` project directory from lesson 09:

.greeter/t/greeter-tests.asd
[source,lisp]
----
(asdf:defsystem :greeter/tests
  :description "Tests for the greeter system"
  :depends-on (:greeter :fiveam)
  :serial t
  :components ((:file "t/suite")
               (:file "t/test-greet")))
----

.greeter/t/suite.lisp
[source,lisp]
----
(in-package :cl-user)

(defpackage :greeter.tests
  (:use :cl :fiveam)
  (:export #:run-all))

(in-package :greeter.tests)

(def-suite greeter-suite
  :description "All greeter tests")

(defun run-all ()
  (run! 'greeter-suite))
----

.greeter/t/test-greet.lisp
[source,lisp]
----
(in-package :greeter.tests)

(in-suite greeter-suite)

;;; --- greet function ---

(test greet-returns-string
  "greet always returns a string."
  (is (stringp (greeter:greet "Alice")))
  (is (stringp (greeter:greet "")))
  (is (stringp (greeter:greet "42"))))

(test greet-formats-name
  "greet embeds the name in the output."
  (is (string= "Hello, Alice!" (greeter:greet "Alice")))
  (is (string= "Hello, Bob!"   (greeter:greet "Bob")))
  (is (string= "Hello, World!" (greeter:greet "World"))))

(test greet-empty-name
  "greet handles an empty string without error."
  (finishes (greeter:greet ""))
  (is (string= "Hello, !" (greeter:greet ""))))

(test greet-special-characters
  "greet preserves special characters in the name."
  (is (string= "Hello, O'Brien!"  (greeter:greet "O'Brien")))
  (is (string= "Hello, José!"     (greeter:greet "José")))
  (is (string= "Hello, <Alice>!"  (greeter:greet "<Alice>"))))

(test greet-long-name
  "greet handles a long name without error."
  (let ((long-name (make-string 1000 :initial-element #\A)))
    (finishes (greeter:greet long-name))
    (is (search long-name (greeter:greet long-name)))))
----

Update `greeter.asd` to declare the test operation:

.greeter/greeter.asd (updated)
[source,lisp]
----
(asdf:defsystem :greeter
  :description "A simple CLI greeter"
  :version "0.1.0"
  :serial t
  :components ((:file "src/packages")
               (:file "src/greeter")
               (:file "src/main"))
  :in-order-to ((asdf:test-op
                 (asdf:load-op :greeter/tests))))

(defmethod asdf:perform ((op asdf:test-op)
                         (system (eql (asdf:find-system :greeter))))
  (uiop:symbol-call :greeter.tests :run-all))
----

'''

=== 7. Running Tests Through ASDF

With the `.asd` wired up, you can run the full test suite via ASDF's test protocol:

[source,lisp]
----
;; Register and test in one step
(pushnew #p"/app/" asdf:*central-registry* :test #'equal)
(asdf:test-system :greeter)
----

This loads `:greeter/tests` (compiling if needed), then calls `perform` on `test-op`, which invokes `greeter.tests:run-all`.

Alternatively, load and run manually:

[source,lisp]
----
(ql:quickload :greeter/tests)
(greeter.tests:run-all)
----

'''

=== 8. The Fix-and-Re-Run Cycle

This is the real payoff of REPL testing. A typical workflow:

. A test fails — Conjure log shows which check and why.
. Open `src/greeter.lisp` in Neovim, fix the function.
. Evaluate the fixed function form with `,ee` (or reload the whole file with `,eb`).
. Call `(greeter.tests:run-all)` with `,ee` in your scratch buffer.
. Repeat until green.

No restart. No external process. No waiting for compilation.

[source,lisp]
----
;; Introduce a deliberate bug:
(defun greet (name)
  (format nil "Hi, ~A?" name))   ; wrong punctuation

;; Run tests — they will fail
(greeter.tests:run-all)

;; Fix the bug (in the real file, reload with ,eb):
(defun greet (name)
  (format nil "Hello, ~A!" name))

;; Re-run — back to green
(greeter.tests:run-all)
----

*Try it:* introduce the bug, evaluate with `,ee`, run tests, fix, run again.
Watch the failure message tell you exactly which form failed and what value it produced.

'''

=== 9. Focusing on a Single Test

When debugging a specific failure, run just one test by name:

[source,lisp]
----
;; Run a single test by name
(fiveam:run! 'greeter.tests::greet-special-characters)
----

Or run all tests in a suite interactively:

[source,lisp]
----
(fiveam:run! 'greeter.tests::greeter-suite)
----
