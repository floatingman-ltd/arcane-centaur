= Appendix F — Commercial Implementations

____
*Series:* xref:learning/lisp/index.adoc[Index] · *Appendix F: Commercial Implementations* ← you are here
____

Two commercial Common Lisp implementations have long histories in production use: *Allegro CL* (Franz Inc.)
and *LispWorks*. Neither is containerized in this series — their licenses prohibit redistribution of
Docker images. Both offer evaluation or free-tier licenses for local use, and both speak the Swank
wire protocol, so Conjure connects to them the same way as SBCL.

'''

== Appendix F — Commercial Implementations

=== Allegro CL (Franz Inc.)

Allegro CL is the oldest continuously developed Common Lisp implementation still in active commercial use.
Franz Inc. has shipped Allegro since the late 1980s.

*Download*: https://franz.com/downloads/[franz.com/downloads]

*Editions:*

[cols="2,5",options="header"]
|===
| Edition | Notes

| Express (free)
| Available for download without charge. Restricted: no source access, limited heap size (512 MB on most platforms), no ACL-style multi-processing module. Suitable for learning and ANSI CL development.

| Professional / Enterprise
| Full heap, all modules, source license, commercial support. Priced per developer seat.
|===

*Strengths:*

* *AllegroServe*: Franz's own HTTP server, mature and production-tested. Ships as an optional module.
* *ELI (Emacs-Lisp Interface)*: the reference Emacs integration for Allegro. Predates SLIME; still widely used by Allegro shops.
* *Long production history*: financial, AI, and government systems that have run Allegro for decades still run on it. Upgrade paths are maintained carefully.
* *Prolog integration*: Allegro includes AllegroProlog for constraint logic programming alongside Common Lisp.
* *Excellent profiler and debugger*: `prof:with-profiling`, the Allegro IDE debugger, and the Graph API for heap inspection are industry-leading.

*Swank and Conjure:*

Allegro ships its own Swank back-end as part of SLIME. To start a Swank server in Allegro:

[source,lisp]
----
;; In Allegro — load Swank (ships with SLIME or can be loaded from Quicklisp)
(require :asdf)
(ql:quickload :swank)

;; Start the server
(swank:create-server :port 4005
                     :style :spawn
                     :dont-close t
                     :interface "127.0.0.1")
----

Then connect in Neovim:

....
,cc   → 127.0.0.1:4005
....

All Conjure evaluation commands (`,ee`, `,eb`, `,er`) work identically.
The Conjure HUD will report `Allegro CL x.x ...` in the connection banner.

*Note on the Express edition heap limit:* the 512 MB cap is generous for learning and most development tasks.
Production workloads with large in-memory datasets require the Professional edition.

'''

=== LispWorks

LispWorks is the other major commercial implementation, developed by LispWorks Ltd (formerly Harlequin).
It is particularly distinguished by its integrated GUI toolkit and its *Deliver* tool for producing small standalone executables.

*Download*: https://www.lispworks.com/downloads/[lispworks.com/downloads]

*Editions:*

[cols="2,5",options="header"]
|===
| Edition | Notes

| Personal (free)
| Free for non-commercial use. Limitations: session timeout (5 hours), heap size cap, no Deliver tool, no multiprocessing on some platforms.

| Professional / Enterprise
| Full access to CAPI, Deliver, multiprocessing, and support. Annual or perpetual licensing.
|===

*Strengths:*

* *CAPI (Common Application Programming Interface)*: LispWorks's cross-platform GUI toolkit. Write one CAPI application; run it on macOS, Windows, Linux, and iOS. No equivalent exists in the open-source CL world.
* *Deliver*: LispWorks's tool for producing small, stripped standalone executables by tree-shaking the Lisp image down to only what is reachable. Delivers of less than 10 MB are common for GUI applications.
* *Excellent documentation*: the LispWorks manual is comprehensive, well-indexed, and accurate.
* *KnowledgeWorks*: a production rule engine (forward/backward chaining) integrated with Common Lisp. Used in expert systems and AI applications.
* *Wide industry adoption*: medical, aerospace, and financial applications that have been in production for 20+ years.

*Swank and Conjure:*

LispWorks supports Swank through SLIME's `swank-lispworks.lisp` backend. Load it at startup:

[source,lisp]
----
;; In LispWorks — load Swank
;; (Assumes SLIME is on the ASDF source registry, or Quicklisp is loaded)
(ql:quickload :swank)

;; Required for Conjure compatibility — disable the dedicated output stream
;; which does not interoperate well with some Conjure Swank clients
(setf swank:*use-dedicated-output-stream* nil)

(swank:create-server :port 4005
                     :style :spawn
                     :dont-close t
                     :interface "0.0.0.0")
----

Then connect Conjure:

....
,cc   → 127.0.0.1:4005
....

*Note on `*use-dedicated-output-stream*`*: setting this to `nil` routes all output through the main Swank channel. This avoids a race condition between the dedicated stream and the main socket that can cause garbled output in the Conjure log.

'''

=== The Swank Wire Protocol — Common to All

Both Allegro CL and LispWorks implement the Swank wire protocol (the same protocol SLIME uses).
The `swank-start.lisp` pattern — `ql:quickload :swank`, then `swank:create-server` — works for both, with minor variation in the backend file that gets loaded automatically when you `(require :swank)` or `(ql:quickload :swank)`.

Conjure's `set g:conjure#client#lisp#swank#connection` settings apply identically:

[source,lua]
----
-- In init.lua or a plugin spec — same settings work for all Swank implementations
vim.g["conjure#client#lisp#swank#connection"] = {
  host = "127.0.0.1",
  port = 4005,
}
----

'''

=== Evaluation Licenses and Local Use

Both implementations permit local evaluation use:

* *Allegro Express*: download without registration; perpetual for non-commercial use within the heap limit.
* *LispWorks Personal*: download without registration; perpetual for non-commercial use within the session and heap limits.

Neither permits redistribution of binaries or Docker images — hence the absence of containers in this series.

If you need to evaluate either implementation, download it from the vendor's site, install it on your host machine, start a Swank server locally (as shown above), and connect Conjure. The learning series lessons work unchanged.

=== When to Consider a Commercial Implementation

[cols="2,5",options="header"]
|===
| Situation | Recommendation

| GUI desktop application with native platform look-and-feel
| LispWorks (CAPI)

| Very small standalone deployment binary
| LispWorks (Deliver) or ECL (compile-through-C)

| Prolog/constraint integration with Lisp
| Allegro CL (AllegroProlog)

| Existing Allegro or LispWorks codebase at work
| Match the existing implementation; migration is nontrivial

| Greenfield server-side or CLI application
| SBCL (best performance and ecosystem); no commercial license needed
|===
