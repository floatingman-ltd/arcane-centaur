This is the capstone lesson. You will build a small but complete HTTP service that:

* Accepts `GET /api/greet?name=Alice` requests
* Returns `{"message":"Hello, Alice!"}` as JSON
* Runs inside the Swank container on port 8080
* Is reachable from your host machine

The libraries are *Hunchentoot* (the most widely deployed Common Lisp HTTP server) and *Jonathan* (a fast JSON encoder/decoder). Both are available from Quicklisp.

Start the SBCL container if it is not running:

[source,sh]
----
LISP_DIR=$PWD docker compose \
  -f ~/.config/nvim/docker/lisp-swank/docker-compose.yml \
  up -d --wait
----

Open a scratch file and connect:

[source,sh]
----
nvim scratch.lisp

=== 2. A Minimal Acceptor

Hunchentoot's `easy-acceptor` is the simplest entry point. It binds a port, serves requests, and calls handler functions you define:

[source,lisp]
----
;; Create an acceptor on port 8080
;; The container's docker-compose.yml publishes 8080 → 8080 on the host
(defvar *server*
  (make-instance 'hunchentoot:easy-acceptor :port 8080))
----

Start it:

[source,lisp]
----
(hunchentoot:start *server*)
; => #<EASY-ACCEPTOR (running) :port 8080>
----

The server is now listening. Any request to `http://127.0.0.1:8080/` from the host will hit Hunchentoot.
The default response to unknown paths is a 404 HTML page — you will define routes next.

'''

=== 3. Defining Routes with define-easy-handler

`hunchentoot:define-easy-handler` registers a function as the handler for a URL path:

[source,lisp]
----
;; Basic route — plain text
(hunchentoot:define-easy-handler (hello-route :uri "/hello") ()
  (setf (hunchentoot:content-type*) "text/plain")
  "Hello from Hunchentoot!")
----

Test it from the host:

[source,sh]
----
curl http://127.0.0.1:8080/hello
# Hello from Hunchentoot!
----

The handler macro signature is: `(define-easy-handler (fn-name :uri path) parameter-specs &body body)`.

Query parameters are extracted via `hunchentoot:parameter`:

[source,lisp]
----
;; Route with a query parameter
(hunchentoot:define-easy-handler (name-route :uri "/hello-name") ()
  (let ((name (hunchentoot:parameter "name")))
    (setf (hunchentoot:content-type*) "text/plain")
    (format nil "Hello, ~A!" (or name "stranger"))))
----

[source,sh]
----
curl "http://127.0.0.1:8080/hello-name?name=Carol"
# Hello, Carol!

curl "http://127.0.0.1:8080/hello-name"
# Hello, stranger!
----

'''

=== 4. Returning JSON

Jonathan encodes a property list or hash table to a JSON string:

[source,lisp]
----
;; Jonathan encoding
(jonathan:to-json '(:message "Hello, Alice!"))
; => "{\"message\":\"Hello, Alice!\"}"

(jonathan:to-json '(:status "ok" :count 3 :items ("a" "b" "c")))
; => "{\"status\":\"ok\",\"count\":3,\"items\":[\"a\",\"b\",\"c\"]}"
----

*Try it:* evaluate each form with `,ee` and inspect the string in the Conjure log.

Set the response content-type to `application/json` and return the encoded string:

[source,lisp]
----
;; A small helper to set JSON content-type and return encoded data
(defun json-response (data)
  (setf (hunchentoot:content-type*) "application/json")
  (jonathan:to-json data))
----

'''

=== 5. The Complete Greet Endpoint

[source,lisp]
----
;; GET /api/greet?name=Alice → {"message":"Hello, Alice!"}
(hunchentoot:define-easy-handler (api-greet :uri "/api/greet") ()
  (let* ((name    (hunchentoot:parameter "name"))
         (safe    (or name "World"))
         (message (format nil "Hello, ~A!" safe)))
    (json-response (list :message message))))
----

Test it:

[source,sh]
----
curl "http://127.0.0.1:8080/api/greet?name=Alice"
# {"message":"Hello, Alice!"}

curl "http://127.0.0.1:8080/api/greet?name=World"
# {"message":"Hello, World!"}

curl "http://127.0.0.1:8080/api/greet"
# {"message":"Hello, World!"}
----

*Try it:* evaluate the handler definition with `,ee`, then run the `curl` commands in a terminal on the host.

'''

=== 6. Complete Runnable Example

This block contains everything you need to start a working service from scratch.
Evaluate the whole buffer with `,eb`:

[source,lisp]
----
;;; web-greeter.lisp — complete, evaluable top to bottom

(ql:quickload '(:hunchentoot :jonathan) :silent t)

;;; ---- Utilities ----

(defun json-response (data)
  "Set Content-Type to application/json and return DATA encoded as JSON."
  (setf (hunchentoot:content-type*) "application/json")
  (jonathan:to-json data))

;;; ---- Routes ----

(hunchentoot:define-easy-handler (api-greet :uri "/api/greet") ()
  "Return a JSON greeting. Query param: name (default World)."
  (let* ((name    (or (hunchentoot:parameter "name") "World"))
         (message (format nil "Hello, ~A!" name)))
    (json-response (list :message message))))

(hunchentoot:define-easy-handler (api-health :uri "/api/health") ()
  "Simple health-check endpoint."
  (json-response '(:status "ok")))

;;; ---- Server lifecycle ----

(defvar *server* nil
  "The running Hunchentoot acceptor, or NIL if stopped.")

(defun start-server (&key (port 8080))
  "Start the HTTP server on PORT (default 8080)."
  (when *server*
    (error "Server is already running on port ~A. Call STOP-SERVER first."
           (hunchentoot:acceptor-port *server*)))
  (setf *server* (make-instance 'hunchentoot:easy-acceptor :port port))
  (hunchentoot:start *server*)
  (format t "~&Server started on port ~A~%" port)
  *server*)

(defun stop-server ()
  "Stop the running HTTP server."
  (unless *server*
    (error "No server is running."))
  (hunchentoot:stop *server*)
  (setf *server* nil)
  (format t "~&Server stopped.~%")
  nil)

;;; ---- Start ----

(start-server :port 8080)
----

After evaluating, the Conjure log will print:

....
Server started on port 8080
....

Verify from the host:

[source,sh]
----
curl -s "http://127.0.0.1:8080/api/greet?name=World"
# {"message":"Hello, World!"}

curl -s "http://127.0.0.1:8080/api/health"
# {"status":"ok"}
----

'''

=== 7. Modifying the Service Without Restarting

One of Hunchentoot's great strengths is that handlers are live Lisp functions.
Redefine one and the change takes effect immediately — no restart needed:

[source,lisp]
----
;; Update the greet endpoint to include a timestamp
(hunchentoot:define-easy-handler (api-greet :uri "/api/greet") ()
  (let* ((name    (or (hunchentoot:parameter "name") "World"))
         (message (format nil "Hello, ~A!" name))
         (ts      (get-universal-time)))
    (json-response (list :message message :timestamp ts))))
----

[source,lisp]
----
;; Evaluate the above with ,ee, then immediately:
;; curl "http://127.0.0.1:8080/api/greet?name=Alice"
;; => {"message":"Hello, Alice!","timestamp":3929299200}
----

This is the REPL-as-production-environment workflow that Lisp web developers use: the server keeps running while you iterate on handlers.

'''

=== 8. Stopping the Server Gracefully

[source,lisp]
----
(stop-server)
; prints: Server stopped.
; => NIL
----

Hunchentoot waits for in-flight requests to complete before releasing the port.

If you restart the whole Swank image (`docker compose ... restart`) the server variable is gone; start fresh with `(start-server)`.

'''

=== 9. Handling POST Requests and Request Bodies

[source,lisp]
----
;; POST /api/echo — reads JSON body and returns it
(hunchentoot:define-easy-handler (api-echo :uri "/api/echo"
                                            :request-type :post) ()
  (let* ((body     (hunchentoot:raw-post-data :force-text t))
         (parsed   (jonathan:parse body))
         (response (list :echo parsed :length (length body))))
    (json-response response)))
----

[source,sh]
----
curl -s -X POST \
     -H "Content-Type: application/json" \
     -d '{"hello":"world"}' \
     http://127.0.0.1:8080/api/echo
# {"echo":{"hello":"world"},"length":17}
----

'''

=== 10. Clack and Ningle — A Brief Mention

Hunchentoot is direct and widely deployed, but the Clack middleware abstraction (analogous to WSGI or Rack) is popular for larger applications:

* *Clack* (`ql:quickload :clack`) — defines a simple interface between web servers and application handlers; your handler is a function from request to response
* *Ningle* (`ql:quickload :ningle`) — a Sinatra-like routing layer built on Clack; expressive route DSL with method and parameter matching
* *Caveman2* (`ql:quickload :caveman2`) — a fuller framework built on Clack with sessions, templates (Djula), and database integration

The trade-off: Hunchentoot is simpler to start with and has no extra layer to understand. Clack/Ningle scale better to complex routing, middleware stacks, and multi-server backends. For this lesson's scope — a small API — Hunchentoot is the right fit.

[source,lisp]
----
;; A minimal Ningle app, for reference
(ql:quickload '(:ningle :clack))

(defvar *app* (make-instance 'ningle:app))

(setf (ningle:route *app* "/api/greet")
      (lambda (params)
        (let* ((name    (or (cdr (assoc "name" params :test #'string=)) "World"))
               (message (format nil "Hello, ~A!" name)))
          (jonathan:to-json (list :message message)))))

;; Start with Clack's Hunchentoot backend
(defvar *handler*
  (clack:clackup *app* :port 8081))

;; Stop
;; (clack:stop *handler*)
----
