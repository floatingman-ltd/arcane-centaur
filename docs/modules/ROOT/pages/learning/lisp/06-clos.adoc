= 06 — CLOS

____
*Series:* xref:learning/lisp/index.adoc[Index] · xref:learning/lisp/05-condition-system.adoc[05 The Condition System] · *06 CLOS* ← you are here · xref:learning/lisp/07-macros.adoc[07 Macros]
____

The Common Lisp Object System (CLOS) is a powerful, dynamic object system with
multiple dispatch, method combination, and a rich metaobject protocol. Unlike
Java or Python, methods belong to *generic functions* — not to classes. A
generic function dispatches on the runtime types of all its arguments, not just
the first one.

This lesson walks through the CLOS core: defining classes, creating instances,
introspecting them, and building a generic function hierarchy. Evaluate every
block in the Swank REPL as you read.

'''

== Defining classes with defclass

[source,lisp]
----
(defclass animal ()
  ((name
    :initarg  :name
    :initform "unnamed"
    :accessor animal-name
    :documentation "The animal's name.")
   (sound
    :initarg  :sound
    :initform "..."
    :reader   animal-sound
    :documentation "The sound this animal makes."))
  (:documentation "Base class for all animals."))
----

Slot options:

[cols="1,3",options="header"]
|===
| Option | Meaning

| `:initarg :name`
| Keyword accepted by `make-instance` to set this slot

| `:initform "unnamed"`
| Default value if the initarg is not supplied

| `:accessor animal-name`
| Generates both a reader and a writer: `(animal-name obj)` and `(setf (animal-name obj) val)`

| `:reader animal-sound`
| Generates only a reader (no setf)

| `:writer (setf animal-sound)`
| Generates only a writer (rarely used explicitly)

| `:documentation "..."`
| Docstring for the slot — visible via `describe`
|===

'''

== Creating instances with make-instance

[source,lisp]
----
(defparameter *cat* (make-instance 'animal :name "Felix" :sound "meow"))

;; Access a slot through its reader
(animal-name *cat*)    ; => "Felix"
(animal-sound *cat*)   ; => "meow"

;; Modify a slot through its accessor (writer)
(setf (animal-name *cat*) "Garfield")
(animal-name *cat*)    ; => "Garfield"
----

'''

== slot-value and slot-boundp

`slot-value` gives direct slot access by name, bypassing the accessor methods.
This is useful for introspection or when you do not want to go through the
generic function dispatch overhead.

[source,lisp]
----
(slot-value *cat* 'name)    ; => "Garfield"
(slot-value *cat* 'sound)   ; => "meow"

;; Modify directly
(setf (slot-value *cat* 'name) "Tom")
(animal-name *cat*)          ; => "Tom"

;; Check whether a slot has been given a value
(slot-boundp *cat* 'name)   ; => T

;; A slot with no initform and not supplied is unbound
(defclass point ()
  ((x :initarg :x)
   (y :initarg :y)))

(defparameter *p* (make-instance 'point :x 3))
(slot-boundp *p* 'x)   ; => T
(slot-boundp *p* 'y)   ; => NIL  (not supplied, no initform)
----

'''

== Introspecting instances with describe

`describe` on an instance prints its class and all slot values — this is your
primary "inspect" tool in the REPL.

[source,lisp]
----
(describe *cat*)
; Prints something like:
;   #<ANIMAL {10042ABC33}>
;     [standard-object]
;   Slots with :INSTANCE allocation:
;     NAME   = "Tom"
;     SOUND  = "meow"
----

You can also inspect the class itself:

[source,lisp]
----
(describe (find-class 'animal))
; Shows the class name, superclasses, subclasses, and slot definitions
----

'''

== defgeneric: declaring the interface

A generic function is the interface; methods are the implementations.
`defgeneric` declares the function's name, lambda-list, and optional
documentation. You do not have to call `defgeneric` before `defmethod` — SBCL
creates the generic function implicitly — but explicit declaration is good style.

[source,lisp]
----
(defgeneric speak (animal)
  (:documentation "Make the animal produce its sound."))

(defgeneric area (shape)
  (:documentation "Return the area of SHAPE as a float."))
----

'''

== defmethod: specialising on a class

[source,lisp]
----
;; Default method — catches any argument type
(defmethod speak ((a animal))
  (format t "~a says: ~a~%" (animal-name a) (animal-sound a)))

(speak *cat*)
; prints: Tom says: meow
----

Define subclasses and their own methods:

[source,lisp]
----
(defclass dog (animal)
  ((breed :initarg :breed :accessor dog-breed :initform "mutt"))
  (:documentation "A dog, subclass of animal."))

(defmethod speak ((d dog))
  (format t "~a barks: WOOF! (breed: ~a)~%"
          (animal-name d)
          (dog-breed d)))

(defparameter *dog* (make-instance 'dog :name "Rex" :sound "woof" :breed "Labrador"))
(speak *dog*)
; prints: Rex barks: WOOF! (breed: Labrador)

;; The animal method is still accessible via call-next-method
----

'''

== Multiple dispatch

This is the feature that sets CLOS apart. A method can specialise on multiple
arguments simultaneously. The runtime picks the most specific method by
examining the types of *all* the specialised arguments.

[source,lisp]
----
(defclass shape () ())
(defclass circle (shape)
  ((radius :initarg :radius :accessor circle-radius)))
(defclass rectangle (shape)
  ((width  :initarg :width  :accessor rectangle-width)
   (height :initarg :height :accessor rectangle-height)))
(defclass triangle (shape)
  ((base   :initarg :base   :accessor triangle-base)
   (height :initarg :height :accessor triangle-height)))

(defgeneric collide (a b)
  (:documentation "Handle a collision between shape A and shape B."))

(defmethod collide ((a circle) (b circle))
  (format t "Circle-circle collision~%"))

(defmethod collide ((a circle) (b rectangle))
  (format t "Circle-rectangle collision~%"))

(defmethod collide ((a shape) (b shape))
  (format t "Generic shape-shape collision~%"))

(defparameter *c* (make-instance 'circle    :radius 5))
(defparameter *r* (make-instance 'rectangle :width 4 :height 3))
(defparameter *t* (make-instance 'triangle  :base 6 :height 4))

(collide *c* *c*)   ; => Circle-circle collision
(collide *c* *r*)   ; => Circle-rectangle collision
(collide *c* *t*)   ; => Generic shape-shape collision  (no specific method)
(collide *r* *c*)   ; => Generic shape-shape collision  (no (rect,circle) method)
----

'''

== area: the shape hierarchy continued

[source,lisp]
----
(defmethod area ((c circle))
  (* pi (expt (circle-radius c) 2)))

(defmethod area ((r rectangle))
  (* (rectangle-width r) (rectangle-height r)))

(defmethod area ((tri triangle))
  (* 0.5 (triangle-base tri) (triangle-height tri)))

(area *c*)    ; => 78.53981633974483
(area *r*)    ; => 12
(area *t*)    ; => 12.0

;; mapcar works because area is a regular function
(mapcar #'area (list *c* *r* *t*))
; => (78.53981633974483 12 12.0)
----

'''

== Method combination: :before, :after, :around

Standard method combination runs all matching `:before` methods (most specific
first), then the primary method, then all `:after` methods (least specific
first). `:around` methods wrap the whole chain and must call `call-next-method`
to let the primary method run.

[source,lisp]
----
(defclass logged-shape (shape) ()
  (:documentation "A shape that logs area computations."))

(defclass logged-circle (circle logged-shape) ())

(defmethod area :before ((s logged-shape))
  (format t "[before] Computing area of ~a~%" (class-name (class-of s))))

(defmethod area :after ((s logged-shape))
  (format t "[after]  Area computed.~%"))

(defmethod area :around ((s logged-shape))
  (format t "[around] Starting...~%")
  (let ((result (call-next-method)))
    (format t "[around] Done, result=~a~%" result)
    result))

(defparameter *lc* (make-instance 'logged-circle :radius 3))
(area *lc*)
; [around] Starting...
; [before] Computing area of LOGGED-CIRCLE
; [after]  Area computed.
; [around] Done, result=28.274333882308138
; => 28.274333882308138
----

`call-next-method` in a primary method calls the next most specific primary
method (enabling cooperative inheritance):

[source,lisp]
----
(defmethod speak :before ((d dog))
  (format t "(clearing throat) "))

(speak *dog*)
; prints: (clearing throat) Rex barks: WOOF! (breed: Labrador)
----

'''

== Inheritance and method resolution order

[source,lisp]
----
;; A class can have multiple direct superclasses (mixins)
(defclass flyable ()
  ()
  (:documentation "Mixin for things that can fly."))

(defgeneric fly (thing)
  (:documentation "Make THING fly."))

(defmethod fly ((f flyable))
  (format t "~a is flying!~%" (class-name (class-of f))))

(defclass flying-cat (animal flyable) ())

(defparameter *fcat*
  (make-instance 'flying-cat :name "Nyan" :sound "nyan"))

(speak *fcat*)   ; => dispatches to (defmethod speak ((a animal)) ...)
(fly   *fcat*)   ; => dispatches to (defmethod fly ((f flyable)) ...)
; "Nyan is flying!"

;; Inspect the class precedence list (C3 linearisation)
(class-precedence-list (find-class 'flying-cat))
; => (#<STANDARD-CLASS FLYING-CAT> #<STANDARD-CLASS ANIMAL>
;     #<STANDARD-CLASS FLYABLE> #<STANDARD-CLASS STANDARD-OBJECT>
;     #<CLASS T>)
----

'''

== print-object: readable REPL output

By default, SBCL prints instances as `#<CLASS-NAME {address}>`. Override
`print-object` to get useful REPL feedback:

[source,lisp]
----
(defmethod print-object ((a animal) stream)
  (print-unreadable-object (a stream :type t)
    (format stream "~a" (animal-name a))))

*cat*    ; => #<ANIMAL Tom>  (much nicer than the raw pointer)
*dog*    ; => #<DOG Rex>

(defmethod print-object ((c circle) stream)
  (print-unreadable-object (c stream :type t)
    (format stream "r=~a" (circle-radius c))))

*c*      ; => #<CIRCLE r=5>
----

`print-unreadable-object` wraps the output in `#<...>` and optionally prints
the type name (`:type t`) and object identity (`:identity t`).

'''

== Putting it together: a complete mini-example

[source,lisp]
----
;;; Shape hierarchy with area, perimeter, and describe output

(defclass shape ()
  ((colour :initarg :colour :accessor shape-colour :initform :grey))
  (:documentation "2D geometric shape."))

(defmethod print-object ((s shape) stream)
  (print-unreadable-object (s stream :type t)
    (format stream "colour=~a" (shape-colour s))))

(defgeneric area (s)      (:documentation "Area of S."))
(defgeneric perimeter (s) (:documentation "Perimeter of S."))
(defgeneric describe-shape (s) (:documentation "Print a human description."))

(defclass circle (shape)
  ((radius :initarg :radius :accessor circle-radius :initform 1.0)))

(defmethod area ((c circle))
  (* pi (expt (circle-radius c) 2)))

(defmethod perimeter ((c circle))
  (* 2 pi (circle-radius c)))

(defmethod describe-shape ((c circle))
  (format t "Circle  colour=~a  radius=~,2f  area=~,2f  perimeter=~,2f~%"
          (shape-colour c) (circle-radius c) (area c) (perimeter c)))

(defclass rectangle (shape)
  ((width  :initarg :width  :accessor rectangle-width  :initform 1.0)
   (height :initarg :height :accessor rectangle-height :initform 1.0)))

(defmethod area ((r rectangle))
  (* (rectangle-width r) (rectangle-height r)))

(defmethod perimeter ((r rectangle))
  (* 2 (+ (rectangle-width r) (rectangle-height r))))

(defmethod describe-shape ((r rectangle))
  (format t "Rectangle colour=~a  ~,2f×~,2f  area=~,2f  perimeter=~,2f~%"
          (shape-colour r) (rectangle-width r) (rectangle-height r)
          (area r) (perimeter r)))

;;; Exercise — evaluate these with ,er then ,ee on each call:

(defparameter *shapes*
  (list (make-instance 'circle    :radius 5  :colour :red)
        (make-instance 'rectangle :width 4 :height 3 :colour :blue)
        (make-instance 'circle    :radius 1  :colour :green)))

;; Describe all shapes
(mapc #'describe-shape *shapes*)
; Circle    colour=RED    radius=5.00   area=78.54  perimeter=31.42
; Rectangle colour=BLUE   4.00×3.00     area=12.00  perimeter=14.00
; Circle    colour=GREEN  radius=1.00   area=3.14   perimeter=6.28

;; Total area
(reduce #'+ (mapcar #'area *shapes*))
; => 93.67433882308138  (approximately)

;; Largest shape by area
(reduce (lambda (a b) (if (>= (area a) (area b)) a b)) *shapes*)
; => #<CIRCLE colour=RED>
----
