= 03 — Forms, Functions & Data

____
*Series:* xref:learning/lisp/index.adoc[Index] · xref:learning/lisp/02-repl-as-ide.adoc[02 The REPL as Your IDE] · *03 Forms, Functions & Data* ← you are here · xref:learning/lisp/04-loop-iteration.adoc[04 LOOP & Iteration]
____

Common Lisp has a rich set of built-in data types and a flexible function
definition syntax. This lesson covers every type you will encounter day-to-day,
plus the forms that create and manipulate them. Evaluate every block as you
read — the REPL output is the explanation.

'''

== Atoms

=== Numbers

[source,lisp]
----
42          ; integer (arbitrary precision — no overflow)
-7          ; negative integer
3.14        ; double-float (default for literals with a decimal point)
1.0e10      ; scientific notation: 10 billion
3/4         ; ratio — exact rational arithmetic, never rounded
(+ 1/3 1/6) ; => 1/2  (not 0.5)
(/ 10 3)    ; => 10/3  (exact)
(/ 10.0 3)  ; => 3.3333333333333335  (float because 10.0 is a float)
----

=== Characters

[source,lisp]
----
#\a         ; the character 'a'
#\A         ; the character 'A' — distinct from #\a
#\Space     ; space character
#\Newline   ; newline character
#\Tab       ; tab character
(char= #\a #\a)     ; => T
(char< #\a #\b)     ; => T
(char-upcase #\a)   ; => #\A
----

=== Strings

[source,lisp]
----
"hello, world"
(length "hello")        ; => 5
(string-upcase "hello") ; => "HELLO"
(string-downcase "HELLO") ; => "hello"
(concatenate 'string "foo" "bar") ; => "foobar"
(subseq "hello" 1 3)   ; => "el"   (start inclusive, end exclusive)
(char "hello" 0)        ; => #\h   (character at index)
(search "ell" "hello")  ; => 1    (index of first match, or NIL)
(string= "abc" "abc")   ; => T
(string< "abc" "abd")   ; => T    (lexicographic comparison)
----

=== Symbols and keywords

Symbols are interned names. They evaluate to their value binding in the current
environment.

[source,lisp]
----
'foo           ; the symbol FOO (quote prevents evaluation)
'cl-user::bar  ; symbol in the CL-USER package — two colons for internal
'cl:car        ; symbol in the CL package — one colon for exported

:keyword       ; a keyword symbol — always evaluates to itself
:host          ; another keyword — used as named arguments and map keys
(eq :foo :foo) ; => T  (keywords are eq-comparable)
----

=== Boolean and nil

[source,lisp]
----
t              ; the canonical true value
nil            ; false and the empty list — the same object

(if t "yes" "no")    ; => "yes"
(if nil "yes" "no")  ; => "no"
(if 0 "yes" "no")    ; => "yes"  — only NIL is false!
(if "" "yes" "no")   ; => "yes"  — empty string is also truthy
----

== Lists and cons cells

Lists are built from cons cells: each cell holds a value (`car`) and a pointer
to the next cell (`cdr`). The last `cdr` is `nil`.

[source,lisp]
----
'(1 2 3)               ; a quoted literal list
(list 1 2 3)           ; same list, constructed at runtime
(cons 1 '(2 3))        ; => (1 2 3)  — prepend 1
(cons 1 (cons 2 (cons 3 nil))) ; => (1 2 3)  — manual construction

(car '(1 2 3))         ; => 1   (first element)
(cdr '(1 2 3))         ; => (2 3)  (everything after the first)
(cadr '(1 2 3))        ; => 2   (car of cdr — second element)
(caddr '(1 2 3))       ; => 3   (third element)
(first '(a b c))       ; => A   (alias for car)
(rest '(a b c))        ; => (B C)  (alias for cdr)
(last '(1 2 3))        ; => (3)  — returns a list containing the last element
(nth 2 '(a b c d))    ; => C   (zero-indexed)
(length '(1 2 3))      ; => 3
(append '(1 2) '(3 4)) ; => (1 2 3 4)  (non-destructive)
(reverse '(1 2 3))     ; => (3 2 1)    (non-destructive)
(member 2 '(1 2 3))    ; => (2 3)  — the tail starting at 2, or NIL
----

A dotted pair is a cons where the `cdr` is not a list:

[source,lisp]
----
(cons 1 2)    ; => (1 . 2)   — dotted pair
(car (cons 1 2))  ; => 1
(cdr (cons 1 2))  ; => 2
----

== Vectors

Vectors are one-dimensional arrays with O(1) random access.

[source,lisp]
----
#(1 2 3)                         ; literal vector (not adjustable)
(make-array 5)                   ; => #(0 0 0 0 0) or similar
(make-array 5 :initial-element 0) ; => #(0 0 0 0 0)
(make-array 3 :initial-contents '(a b c)) ; => #(A B C)

(aref #(10 20 30) 1)             ; => 20  (zero-indexed)

;; Adjustable vector — you can push to it
(defparameter *v* (make-array 0 :adjustable t :fill-pointer 0))
(vector-push-extend 'a *v*)      ; => 0  (index of pushed element)
(vector-push-extend 'b *v*)      ; => 1
(vector-push-extend 'c *v*)      ; => 2
*v*                              ; => #(A B C)
(length *v*)                     ; => 3
(aref *v* 0)                     ; => A

;; setf on aref is the mutation idiom
(setf (aref *v* 0) 'z)
*v*                              ; => #(Z B C)
----

'''

== Hash tables

Hash tables are the standard mutable key-value store. The default test is `eql`,
which works for numbers, characters, and symbols. Use `:test #'equal` for
string keys.

[source,lisp]
----
(defparameter *h* (make-hash-table))
(setf (gethash 'a *h*) 1)
(setf (gethash 'b *h*) 2)
(setf (gethash 'c *h*) 3)

(gethash 'a *h*)     ; => 1, T     (value and a found-p flag)
(gethash 'z *h*)     ; => NIL, NIL (not found)
(gethash 'z *h* 99)  ; => 99, NIL  (default value when not found)

(remhash 'b *h*)     ; => T   (removed; returns NIL if key was absent)
(hash-table-count *h*)  ; => 2

;; Iterate over all entries
(maphash (lambda (k v) (format t "~a => ~a~%" k v)) *h*)

;; String keys — must use :test #'equal
(defparameter *names* (make-hash-table :test #'equal))
(setf (gethash "alice" *names*) 42)
(gethash "alice" *names*)   ; => 42, T
----

== defun: function definition

=== Required arguments

[source,lisp]
----
(defun add (x y)
  "Add two numbers."
  (+ x y))

(add 3 4)   ; => 7
----

=== Optional arguments

[source,lisp]
----
(defun greet (name &optional (greeting "Hello"))
  "Greet NAME with an optional custom GREETING."
  (format nil "~a, ~a!" greeting name))

(greet "Walt")            ; => "Hello, Walt!"
(greet "Walt" "Howdy")   ; => "Howdy, Walt!"
----

Each optional arg can have a default and a supplied-p variable:

[source,lisp]
----
(defun connect (&optional (host "127.0.0.1" host-supplied-p)
                           (port 4005 port-supplied-p))
  (list host host-supplied-p port port-supplied-p))

(connect)                      ; => ("127.0.0.1" NIL 4005 NIL)
(connect "10.0.0.1")           ; => ("10.0.0.1" T 4005 NIL)
(connect "10.0.0.1" 9000)      ; => ("10.0.0.1" T 9000 T)
----

=== Keyword arguments

[source,lisp]
----
(defun make-point (&key (x 0) (y 0) (z 0))
  "Create a 3D point as a list."
  (list :x x :y y :z z))

(make-point)             ; => (:X 0 :Y 0 :Z 0)
(make-point :y 5 :z 3)  ; => (:X 0 :Y 5 :Z 3)
(make-point :x 1 :y 2)  ; => (:X 1 :Y 2 :Z 0)
----

=== Rest arguments

[source,lisp]
----
(defun sum (&rest numbers)
  "Sum any number of arguments."
  (reduce #'+ numbers :initial-value 0))

(sum 1 2 3 4 5)   ; => 15
(sum)             ; => 0
----

=== Mixed argument types

The order is always: required, `&optional`, `&rest`, `&key`.

[source,lisp]
----
(defun format-table (title &rest rows &key (separator " | "))
  ;; Note: mixing &rest and &key means keywords are also in the rest list
  (declare (ignore rows))
  (format nil "~a~%~a" title separator))
----

In practice, mix `&optional` with required, or `&key` with required, but avoid
`&optional` + `&key` together — it leads to confusing semantics.

== Lambda and higher-order functions

[source,lisp]
----
;; An anonymous function
(lambda (x) (* x x))

;; Apply it immediately
((lambda (x) (* x x)) 5)   ; => 25

;; funcall: call a function stored in a variable
(let ((sq (lambda (x) (* x x))))
  (funcall sq 7))           ; => 49

;; apply: call with an argument list
(apply #'+ '(1 2 3 4))     ; => 10
(apply #'max 3 '(1 2 4))   ; => 4  (first args then a list)

;; #' is the function namespace accessor — required for named functions
;; passed as values
(mapcar #'string-upcase '("hello" "world"))
; => ("HELLO" "WORLD")
----

'''

== Multiple return values

[source,lisp]
----
;; values returns multiple values
(values 1 2 3)   ; => 1, 2, 3  (three separate values)

;; floor returns quotient and remainder
(floor 17 5)     ; => 3, 2

;; Bind multiple return values
(multiple-value-bind (quotient remainder)
    (floor 17 5)
  (format t "~a remainder ~a" quotient remainder))
; prints: 3 remainder 2

;; Collect all values into a list
(multiple-value-list (floor 17 5))  ; => (3 2)

;; Only the first value is used in a normal expression context
(+ (floor 17 5) 10)   ; => 13  (3 + 10; remainder 2 is discarded)
----

'''

== Let and let*: local bindings

[source,lisp]
----
;; let: all bindings evaluated in parallel (outer scope)
(let ((x 10)
      (y 20))
  (+ x y))   ; => 30

;; let*: bindings evaluated sequentially — each sees the previous
(let* ((x 10)
       (y (* x 2))   ; y can reference x
       (z (+ x y)))  ; z can reference x and y
  (list x y z))      ; => (10 20 30)

;; let does NOT allow forward references
;; This would signal an error because y is not yet defined:
;; (let ((x y) (y 5)) x)
----

Prefer `let` when bindings are independent (cleaner intent). Use `let*` when
each binding builds on the previous.
