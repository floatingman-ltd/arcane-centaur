= 05 — The Condition System

____
*Series:* xref:learning/lisp/index.adoc[Index] · xref:learning/lisp/04-loop-iteration.adoc[04 LOOP & Iteration] · *05 The Condition System* ← you are here · xref:learning/lisp/06-clos.adoc[06 CLOS]
____

Common Lisp's condition system is the most powerful error-handling mechanism in
any mainstream language. Unlike try/catch in Java or Python, conditions separate
*signalling* a problem from *handling* it, and handlers can run *without
unwinding the call stack*. This means the code that detected the problem can
offer multiple recovery paths (restarts), and calling code higher up the stack
can choose which one to invoke.

This lesson builds from simple error handling up to restarts and the interactive
Swank debugger (SLDB).

'''

== Conditions are objects

A condition is a CLOS instance of a class that inherits from `condition`. The
standard library defines a hierarchy:

....
condition
  ├── serious-condition
  │   ├── error         ← most fatal
  │   └── storage-condition
  └── warning
  └── (simple-condition subclasses)
....

You signal conditions; you do not throw them. Handling is done by functions
registered dynamically in the call stack, not by a syntactic enclosing block.

'''

== Signalling conditions

=== error

`error` signals an `error` condition. If nothing handles it, SBCL enters the
debugger (SLDB via Swank).

[source,lisp]
----
(error "something went wrong")
; Swank enters the debugger:
;   something went wrong
;   [Condition of type SIMPLE-ERROR]

(error "value ~a is out of range [~a, ~a]" 42 0 10)
; => enters debugger with a formatted message
----

=== warn

`warn` signals a `warning`. By default it prints a message to `*error-output*`
and continues — it does not enter the debugger.

[source,lisp]
----
(warn "this is a warning: ~a" "check your config")
; prints to stderr: WARNING: this is a warning: check your config
; => NIL  (execution continues)
----

=== signal

`signal` is the lowest-level signaller. It signals any condition object but
does not invoke the debugger if nothing handles it — it just returns `nil`.

[source,lisp]
----
(signal 'simple-condition :format-control "informational event")
; => NIL  (silently returns if no handler)
----

'''

== handler-case: catching and recovering

`handler-case` is the closest equivalent to try/catch. It evaluates its body
and, if a condition matching one of the clauses is signalled, unwinds the stack
to the `handler-case` form and runs the matching clause.

[source,lisp]
----
(handler-case
    (error "boom")
  (error (c)
    (format t "caught: ~a~%" c)))
; prints: caught: boom
; => NIL

;; Catching specific condition types
(handler-case
    (/ 1 0)
  (division-by-zero (c)
    (format t "division by zero: ~a~%" c)
    :infinity)
  (error (c)
    (format t "other error: ~a~%" c)
    :error))
; => :INFINITY

;; The condition object gives you structured information
(handler-case
    (error 'type-error :datum "foo" :expected-type 'integer)
  (type-error (c)
    (list (type-error-datum c) (type-error-expected-type c))))
; => ("foo" INTEGER)
----

The clauses are tested in order; the first matching class wins. An `error`
clause catches any error; put more specific types first.

'''

== handler-bind: handling without unwinding

`handler-bind` installs a handler that runs *in the dynamic context of the
signal* — the call stack is still intact. This is the key difference from
`handler-case`. A `handler-bind` handler can inspect the condition, log it, or
invoke a restart, then either return (letting the signal propagate) or invoke a
non-local exit.

[source,lisp]
----
;; handler-bind runs the handler but does NOT unwind unless you force it
(handler-bind ((warning (lambda (w)
                          (format t "intercepted warning: ~a~%" w)
                          (muffle-warning w))))   ; suppress the default print
  (warn "this warning is silenced")
  :continued)
; prints: intercepted warning: this warning is silenced
; => :CONTINUED

;; Contrast with handler-case: the body's result is returned if no error
(handler-case
    (* 6 7)
  (error (c) (declare (ignore c)) :error))
; => 42

;; handler-bind without invoking a restart just logs and lets it propagate
(defun try-divide (n d)
  (handler-bind ((division-by-zero
                   (lambda (c)
                     (format t "warning: caught ~a, propagating~%" c))))
    (/ n d)))

;; This still enters the debugger because the handler did not handle it:
;; (try-divide 1 0)

;; To actually recover, invoke a restart or use a non-local exit:
(handler-bind ((division-by-zero
                 (lambda (c)
                   (declare (ignore c))
                   (return-from try-divide 0))))
  (/ 1 0))
; => 0
----

'''

== Restarts: offering recovery options

Restarts are named recovery points established with `restart-case`. The code
that signals the error offers *multiple possible recoveries*; the handler
chooses which one.

=== restart-case

[source,lisp]
----
(defun carefully-divide (n d)
  (restart-case
      (if (zerop d)
          (error 'division-by-zero :operation '/ :operands (list n d))
          (/ n d))
    (return-zero ()
      :report "Return 0 instead of dividing"
      0)
    (return-infinity ()
      :report "Return :infinity"
      :infinity)
    (use-value (v)
      :report "Use a supplied value"
      :interactive (lambda () (list (read)))
      v)))
----

Now callers can choose a restart:

[source,lisp]
----
;; Invoke a named restart from a handler
(handler-bind ((division-by-zero
                 (lambda (c)
                   (declare (ignore c))
                   (invoke-restart 'return-zero))))
  (carefully-divide 10 0))
; => 0

(handler-bind ((division-by-zero
                 (lambda (c)
                   (declare (ignore c))
                   (invoke-restart 'use-value 999))))
  (carefully-divide 10 0))
; => 999
----

=== invoke-restart and find-restart

[source,lisp]
----
;; find-restart returns the restart object if active, NIL if not
(restart-case
    (let ((r (find-restart 'return-zero)))
      (if r
          (invoke-restart r)
          :no-restart))
  (return-zero () 42))
; => 42
----

=== with-simple-restart

A convenience macro for the common case of a single restart:

[source,lisp]
----
(defun read-integer (prompt)
  (with-simple-restart (skip "Skip this input and return NIL")
    (format t "~a: " prompt)
    (let ((val (read)))
      (unless (integerp val)
        (error "~a is not an integer" val))
      val)))
----

=== ignore-errors

The simplest recovery — silently ignore any error and return `nil`:

[source,lisp]
----
(ignore-errors (/ 1 0))
; => NIL, #<DIVISION-BY-ZERO ...>  (two values: result and condition)

(ignore-errors (+ 1 2))
; => 3, NIL  (no error — condition is NIL)
----

Use `multiple-value-bind` to distinguish the error case:

[source,lisp]
----
(multiple-value-bind (result condition)
    (ignore-errors (parse-integer "not-a-number"))
  (if condition
      (format t "parse failed: ~a~%" condition)
      (format t "parsed: ~a~%" result)))
; prints: parse failed: ...
----

'''

== Defining custom conditions

Use `define-condition` to create condition classes with typed slots. This is
the idiomatic way to carry structured error information.

[source,lisp]
----
(define-condition file-missing-error (error)
  ((path :initarg :path
         :reader file-missing-path))
  (:report (lambda (c stream)
             (format stream "File not found: ~a" (file-missing-path c)))))

;; Signal it with structured data
(defun load-config (path)
  (unless (probe-file path)
    (error 'file-missing-error :path path))
  :loaded)

;; Handle it with typed access to the slot
(handler-case (load-config "/tmp/no-such-file.conf")
  (file-missing-error (c)
    (format t "caught: ~a~%" (file-missing-path c))
    :default-config))
; prints: caught: /tmp/no-such-file.conf
; => :DEFAULT-CONFIG
----

'''

== The Swank debugger (SLDB)

When a condition reaches Swank unhandled, SBCL enters the Swank interactive
debugger. Conjure displays it in a special buffer. Here is what you will see:

....
;; An unhandled DIVISION-BY-ZERO condition appears as:
;;
;;   Division by zero
;;   [Condition of type DIVISION-BY-ZERO]
;;
;;   Restarts:
;;    0: [RETRY] Retry SLIME evaluation request.
;;    1: [*ABORT] Return to SLIME's top level.
;;    2: [ABORT] abort thread (#<THREAD "repl-thread" RUNNING {10042AEBB3}>)
;;
;;   Backtrace:
;;    0: (SB-KERNEL::INTEGER-DIVIDE-BY-ZERO-ERROR ...)
;;    ...
....

Key actions in the SLDB buffer:

[cols="1,3",options="header"]
|===
| Key | Action

| `q`
| Quit the debugger and abort back to the top level (invokes the `ABORT` restart)

| `0`, `1`, `2` ...
| Invoke the numbered restart

| `Enter`
| Expand a backtrace frame

| `,cs` then `,cc`
| Close and reconnect Conjure if the session is corrupted
|===

To deliberately trigger SLDB for exploration:

[source,lisp]
----
(/ 1 0)   ; evaluate with ,ee — SLDB appears in a Conjure buffer
          ; press q to quit and return to normal editing
----

'''

== Worked example: divide with restarts

This ties everything together: a `divide` function that never crashes, offering
callers two named restarts.

[source,lisp]
----
(define-condition bad-divisor (error)
  ((divisor :initarg :divisor :reader bad-divisor-value))
  (:report (lambda (c s)
             (format s "Divisor ~a is not valid (must be non-zero integer)"
                     (bad-divisor-value c)))))

(defun safe-divide (numerator divisor)
  "Divide NUMERATOR by DIVISOR.
   On zero divisor, offers RETURN-ZERO and USE-VALUE restarts."
  (restart-case
      (progn
        (unless (and (integerp divisor) (not (zerop divisor)))
          (error 'bad-divisor :divisor divisor))
        (/ numerator divisor))
    (return-zero ()
      :report "Return 0 as the result of the division"
      0)
    (use-value (v)
      :report "Supply a replacement divisor"
      (/ numerator v))))

;; Normal use
(safe-divide 10 2)    ; => 5

;; Handler chooses return-zero
(handler-bind ((bad-divisor
                 (lambda (c)
                   (declare (ignore c))
                   (invoke-restart 'return-zero))))
  (safe-divide 10 0))
; => 0

;; Handler supplies a replacement divisor
(handler-bind ((bad-divisor
                 (lambda (c)
                   (declare (ignore c))
                   (invoke-restart 'use-value 5))))
  (safe-divide 10 0))
; => 2

;; Let it reach SLDB interactively (evaluate with ,ee and press q after)
;; (safe-divide 10 0)
----
