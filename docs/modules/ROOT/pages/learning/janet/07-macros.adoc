== 07 — Macros with Janet

Macros let you extend Janet's syntax by transforming code before it is evaluated.
Used well, they eliminate repetition and enable expressive DSLs.
Used poorly, they create hard-to-read, hard-to-debug code.
This lesson covers quasiquoting, `defmacro`, macro expansion, and when to reach for a macro vs a function.

Open a scratch file and the Conjure log before starting:

[source,sh]
----
nvim scratch.janet
----

....
,lv
....

'''''

=== 1. Code Is Data

In Janet, code and data share the same structure — both are tuples, tables, and symbols.
Macros receive unevaluated code as their arguments and return new code to be evaluated in its place.

[source,janet]
----
# This is data (a tuple)
'(+ 1 2)

# This is also data — a nested structure
'(defn greet [name] (string "Hello, " name))

# eval turns data into code
(eval '(+ 1 2))   # => 3
----

*Try it:* evaluate `'(+ 1 2)` with `,ee` — confirm it returns a tuple, not a number.

'''''

=== 2. Quasiquoting

Quasiquoting builds code templates.
A quasiquoted form is like a quoted form, except you can splice in runtime values with `,` (unquote) and `,;` (splice-unquote).

[source,janet]
----
(def x 42)

# quote — x is not evaluated
'(the answer is x)
# => (the answer is x)

# quasiquote with ~ — x is still not evaluated
~(the answer is x)
# => (the answer is x)

# unquote with , — x IS evaluated
~(the answer is ,x)
# => (the answer is 42)
----

Splice-unquote (`,;`) inserts the contents of a sequence, not the sequence itself:

[source,janet]
----
(def items [1 2 3])

~(list ,items)     # => (list (1 2 3))   — items as a single value
~(list ,;items)    # => (list 1 2 3)     — items spliced in
----

*Try it:* evaluate both quasiquoted forms with `,ee` and compare the results.

'''''

=== 3. defmacro

`defmacro` defines a macro.
The body returns a piece of code (a data structure) that replaces the macro call at compile time.

[source,janet]
----
# unless: run body when condition is false
(defmacro unless [condition & body]
  ~(when (not ,condition)
     ,;body))

# Usage — expands before evaluation
(unless false
  (print "this runs"))

(unless true
  (print "this never runs"))
----

*Try it:* evaluate the `defmacro` form, then both `unless` calls with `,ee`.

Another example — `swap!` to exchange two mutable bindings:

[source,janet]
----
(defmacro swap! [a b]
  ~(let [tmp ,a]
     (set ,a ,b)
     (set ,b tmp)))

(var p 1)
(var q 2)
(swap! p q)
[p q]   # => (2 1)
----

'''''

=== 4. macex — Inspecting Expansion

`macex` expands a macro call one level and returns the generated code.
Use it to understand what a macro produces:

[source,janet]
----
(macex '(unless false (print "hi")))
# => (when (not false) (print "hi"))

(macex '(swap! p q))
# => (let (tmp p) (set p q) (set q tmp))
----

*Try it:* wrap any macro call in `macex` and evaluate with `,ee` to see what code it generates.
This is your primary debugging tool for macros.

`macex1` expands only one level; `macex` keeps expanding until no more macros remain:

[source,janet]
----
(macex1 '(unless false (print "hi")))
# => (when (not false) (print "hi"))

(macex '(unless false (print "hi")))
# => (if (not false) (do (print "hi")))
----

'''''

=== 5. When to Use a Macro vs a Function

Most problems should be solved with functions.
Reach for a macro only when a function cannot do the job.

[cols="2,4,4",options="header"]
|===
| Situation | Use | Reason

| Delay evaluation of arguments
| Macro
| Functions always evaluate all arguments before the call

| Introduce new binding forms (`let`-like)
| Macro
| Functions cannot introduce bindings in the caller's scope

| Log with source location / form text
| Macro
| The macro can capture the unevaluated form as a string

| Wrap repeated code pattern with no new syntax
| Function
| Simpler to test, debug, and compose

| Transform a data structure at runtime
| Function
| Macros run at compile time — runtime data is unavailable
|===

Example of a case that *looks* like it needs a macro but does not — a retry helper:

[source,janet]
----
# As a function — works fine because the body is a function
(defn retry [n f]
  (var attempts 0)
  (var done false)
  (var result nil)
  (while (and (not done) (< attempts n))
    (let [[ok val] (protect (f))]
      (if ok
        (do (set result val) (set done true))
        (++ attempts))))
  result)

(retry 3 (fn []
           (print "trying...")
           (if (< (math/random) 0.7)
             (error "unlucky")
             42)))
----

*Try it:* evaluate `retry` with `,er`, then call it with `,ee` a few times — watch the Conjure log.

=== 6. Mini-Project: A Simple Assert Macro

Build an `assert-eq` macro that checks equality and prints a meaningful failure message including the original expressions.

[source,janet]
----
(defmacro assert-eq [expected actual]
  ~(let [e ,expected
         a ,actual]
     (if (= e a)
       (print "PASS")
       (print (string "FAIL: expected " (describe e)
                      " but got " (describe a)
                      " (form: " ,(string actual) ")")))))
----

[source,janet]
----
# Tests
(assert-eq 4 (+ 2 2))        # PASS
(assert-eq 10 (* 3 3))       # FAIL: expected 10 but got 9 (form: (* 3 3))
(assert-eq "hi" (string "h" "i"))  # PASS
----

*Workflow:*

. Evaluate the macro definition with `,er`.
. Evaluate each test with `,ee` and read the output in the Conjure log.
. Use `macex` to inspect what `(assert-eq 4 (+ 2 2))` expands to.
. Add a third argument for a custom message and update the macro.
