== 06 — System Prompts & Context

A system prompt is an instruction that shapes how the model behaves across an entire
conversation — before you ask your first question.
This lesson covers how to set one in Avante, how context windows work, and how to
structure longer multi-turn conversations effectively.

Open Avante with `<leader>ao` before starting.

'''''

=== 1. What a System Prompt Does

Without a system prompt, the model responds as a generic assistant.
A system prompt lets you establish a persona, set an expertise level, restrict the
scope, or define a response format — and have that persist across every message in
the session.

*Try it — without a system prompt:*

Open a fresh Avante session (`<leader>ao`) and ask:

....
What is a monad?
....

Note how detailed and broad the response is.

*Now with a system prompt:* close and reopen Avante, then send this as your *first* message:

....
You are a functional programming tutor. Explain things simply, using Janet or Clojure
examples. Keep responses under five sentences.
....

Then ask:

....
What is a monad?
....

*Confirm you see:* a shorter, Janet/Clojure-flavoured explanation.
The model carries this instruction forward for all subsequent messages in this session.

'''''

=== 2. Useful System Prompt Patterns

[cols="3,5",options="header"]
|===
| Pattern | Example

| Set expertise level
| "Assume I'm an experienced programmer. Skip basic explanations."

| Constrain response length
| "Keep every response under three sentences unless I ask for detail."

| Set a reviewer role
| "You are a code reviewer. Focus on correctness and edge cases, not style."

| Set a language context
| "We are working in a Neovim Lua config. Assume the Neovim 0.11+ API."

| Define output format
| "Always respond in AsciiDoc format with `[source,lua]` code blocks."
|===

*Try it:* open a fresh session and set this system prompt:

....
You are a Neovim Lua expert reviewing a config. Be terse. Flag only genuine problems —
not style preferences. Use one-line responses for simple questions.
....

Then ask:

....
Is it safe to call vim.api.nvim_buf_set_option in an autocmd?
....

*Confirm you see:* a direct, terse response.

'''''

=== 3. Context Windows

Every model has a context window — a maximum number of tokens it can hold at once.
For `llama3.1:8b` this is typically 8,000–128,000 tokens depending on the version.

In practice, this means:

* Long conversations degrade — early messages drop out of the window
* Pasting very large files may exceed the limit
* Ollama will silently truncate the oldest messages when the window fills

*Signs the context window is full:*

* The model "forgets" something it acknowledged earlier
* Responses become less coherent
* Answers to earlier questions conflict with what it said before

*Strategies:*

* Start a fresh session (`<leader>ao`) when switching to a new topic
* Paste only the relevant section of a file, not the whole thing
* Re-state key constraints at the start of new sessions rather than relying on history

'''''

=== 4. Structuring Multi-Turn Conversations

For complex tasks that need multiple exchanges, structure the conversation deliberately:

*Pattern: anchor → explore → conclude*

. *Anchor*: set the system prompt and paste the relevant context
. *Explore*: ask a series of focused questions, building on each response
. *Conclude*: ask for a summary or final recommendation once you have what you need

*Example — architecture review conversation:*

[source,text]
----
[Message 1 — anchor]
You are a Neovim config reviewer. Here is the full lua/config/lsp.lua:

<paste file contents>

[Message 2 — explore]
What servers are configured and what are their roles?

[Message 3 — explore]
Are there any servers that could share an on_attach function?

[Message 4 — conclude]
Summarise the refactoring you'd suggest in three bullet points.
----

*Try it:* run this conversation using your actual `lua/config/lsp.lua`. Adjust the
explore questions to what you're actually curious about.

'''''

=== 5. When to Start Fresh vs Continue

[cols="3,5",options="header"]
|===
| Situation | Recommendation

| Switching to a completely different topic
| Start fresh (`<leader>ao`) — avoids contaminating the new context with unrelated history

| Following up on the same code or question
| Continue the session — the model has the context it needs

| Getting incoherent or contradictory responses
| Start fresh and re-state the key context

| Starting a new code review
| Start fresh with a role-setting system prompt
|===
