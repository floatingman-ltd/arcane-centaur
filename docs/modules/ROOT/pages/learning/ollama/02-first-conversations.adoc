'''''

== 02 — First Conversations

Now that Ollama is running, this lesson walks through the Avante interface in depth:
the buffer layout, sending messages, reviewing conversation history, and switching
between the ollama and copilot backends.

Make sure the Ollama service is running before starting:

[source,sh]
----
curl -s http://127.0.0.1:11434/
----

*Confirm you see:* `Ollama is running`

'''''

=== 1. Opening Avante

Three keymaps open Avante — each serves a different purpose:

[cols="2,5",options="header"]
|===
| Keymap | What it does

| `<leader>ao`
| Switch to *ollama* provider and open Avante (use for local/offline work)

| `<leader>ac`
| Switch to *copilot* provider and open Avante (use when connected and wanting GPT-4-class responses)

| `<leader>aa`
| Open Avante with *whichever provider is currently active* (no switch)
|===

Press `<leader>ao` now.

*Confirm you see:* the Avante panel opens on the right side of your editor.

'''''

=== 2. The Avante Buffer Layout

....
┌─────────────────────┬──────────────────────────┐
│                     │  Avante                  │
│   your file         │  ─────────────────────── │
│                     │  [conversation history]  │
│                     │                          │
│                     │  ─────────────────────── │
│                     │  > input area            │
└─────────────────────┴──────────────────────────┘
....

* *Conversation history* — responses accumulate here across the session
* *Input area* — type your message here, press `Enter` to submit

To close Avante: press `<leader>aa` again (toggle) or `:AvanteToggle`.

'''''

=== 3. Sending Your First Messages

With Avante open and ollama active, type each prompt in the input area and press `Enter`:

*Prompt 1 — capability check:*

....
What programming languages do you know well?
....

*Confirm you see:* a list of languages. Response time on CPU is ~10–30 seconds for an 8B model.

*Prompt 2 — context awareness:*

....
What file am I currently editing?
....

*Confirm you see:* Avante does not automatically know which file you have open unless you
provide context. We'll cover that workflow in lesson 04.

*Prompt 3 — a short task:*

....
Write a one-line Janet function that squares a number.
....

*Confirm you see:* something like `(defn square [x] (* x x))` in the response.

'''''

=== 4. Conversation History

Avante maintains conversation history within a session. Each message you send is in the
context of the previous exchange.

*Try it:* after the square function response, send:

....
Now write a cube function the same way.
....

*Confirm you see:* Avante uses the established style from the previous response — it knows
you want the same Janet pattern without you repeating the context.

To start a fresh conversation: close and reopen Avante with `<leader>ao`.

'''''

=== 5. Switching Providers Mid-Session

You can switch between ollama and copilot at any time:

[source,sh]
----
# From inside Neovim:
<leader>ao   # switch to ollama
<leader>ac   # switch to copilot
----

*Try it:* press `<leader>ac` to switch to the copilot provider.

*Confirm you see:* Avante reopens. The provider indicator in the buffer header changes.

Then ask:

....
What provider are you?
....

*Confirm you see:* a response reflecting the copilot backend (GPT-based).

Switch back to ollama with `<leader>ao` for the rest of this lesson.

'''''

=== 6. Useful Avante Commands

[cols="3,5",options="header"]
|===
| Command / Keymap | What it does

| `<leader>aa`
| Open Avante with current provider (toggle)

| `<leader>ao`
| Switch to ollama + open

| `<leader>ac`
| Switch to copilot + open

| `:AvanteToggle`
| Toggle the Avante panel

| `:AvanteClear`
| Clear conversation history
|===

'''''
