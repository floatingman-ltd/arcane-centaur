Working with Haskell

Haskell support is provided by haskell-tools.nvim, which bundles LSP integration, a GHCi REPL, and Hoogle search into a single plugin.

All Haskell plugins lazy-load only when you open a .hs or .lhs file.

Quick Start

  1. Install GHC and haskell-language-server via ghcup (see Prerequisites).

  2. Open a Haskell source file — plugins load automatically:

    nvim src/Main.hs

  3. Press <leader>rr to open a GHCi REPL for the current package.

  4. Evaluate code in the REPL, or press <leader>rf to open a file-scoped REPL.

LSP

haskell-tools manages the hls LSP connection automatically — no manual lspconfig setup is needed. See the Code Intelligence cheatsheet for shared LSP keybindings (gd, K, gr, etc.).

REPL (GHCi)

haskell-tools provides a built-in GHCi REPL. Keybindings are defined in after/ftplugin/haskell.lua.

Leader is Space; no LocalLeader bindings are used in Haskell buffers.

Keys Action

<leader>rr

Toggle GHCi REPL for the current package

<leader>rf

Toggle GHCi REPL for the current file

<leader>rq

Quit the GHCi REPL

Space-cl

Run code lenses (haskell-language-server)

Space-hs

Hoogle search for type under cursor

Space-ea

Evaluate all code snippets in the buffer

Configuration

Plugin spec lives in lua/plugins/haskell.lua and ftplugin overrides live in after/ftplugin/haskell.lua.
