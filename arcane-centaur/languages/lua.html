Working with Lua

This configuration provides full LSP support and format-on-save for Lua via lua-language-server and stylua.

Quick navigation

Language server setup

Prerequisites

Install lua-language-server and stylua

Quick Start

  1. Open a Lua file — LSP attaches automatically:

    nvim lua/config/lsp.lua

  2. Use LSP keymaps (see Code Intelligence cheatsheet for the full list):

    Keys Action

    gd

    Go to definition

    K

    Hover documentation

    gr

    Find references

    <leader>rn

    Rename symbol

    <leader>ca

    Code action

    <leader>e

    Show diagnostics

  3. Format on save — stylua runs automatically when you write a .lua file. Manual format: <leader>f in normal or visual mode.

LSP

lua_ls (lua-language-server) is configured in lua/config/lsp.lua and attaches automatically when a Lua buffer is opened. It provides completions, diagnostics, go-to-definition, and hover docs.

Verify the server is attached with:

:lua =vim.lsp.get_clients({bufnr=0})

Formatting

stylua runs via conform.nvim on every save (2-second timeout). It also respects a stylua.toml or .stylua.toml in the project root.

To format manually:

Keys Action

<leader>f

Format buffer or visual selection

Indent settings for Lua buffers (set in after/ftplugin/lua.lua): 2-space tabs, spaces only.

Installing lua-language-server

lua-language-server is not available via apt or snap on Ubuntu 24.04. Install from the GitHub releases page.
Platform Method

Ubuntu / Debian

Binary download — see below

Arch Linux

sudo pacman -S lua-language-server

macOS (Homebrew)

brew install lua-language-server

Ubuntu / Debian — binary install

Download and extract the latest release (check releases for the current version):

VERSION=3.18.2
wget https://github.com/LuaLS/lua-language-server/releases/download/${VERSION}/lua-language-server-${VERSION}-linux-x64.tar.gz
mkdir -p ~/.local/opt/lua-language-server
tar -xzf lua-language-server-${VERSION}-linux-x64.tar.gz -C ~/.local/opt/lua-language-server

The binary must run from its extracted location (it resolves scripts relative to itself). Do not symlink the binary directly — create a wrapper script in ~/.local/bin instead (~/.local/bin is already on $PATH on Ubuntu 24.04):

cat > ~/.local/bin/lua-language-server << 'EOF'
#!/bin/sh
exec "$HOME/.local/opt/lua-language-server/bin/lua-language-server" "$@"
EOF
chmod +x ~/.local/bin/lua-language-server

Verify:

lua-language-server --version

Installing stylua

stylua is not available via apt on Ubuntu 24.04. Install the binary directly from GitHub releases: 
curl -sLo /tmp/stylua.zip https://github.com/JohnnyMorganz/StyLua/releases/latest/download/stylua-linux-x86_64.zip
unzip /tmp/stylua.zip -d ~/.local/bin/
chmod +x ~/.local/bin/stylua

Or via package manager on other platforms:

Platform Command

Arch Linux

sudo pacman -S stylua

macOS (Homebrew)

brew install stylua

Verify:

stylua --version

Neovim

Neovim ≥ 0.12 is required. See Getting Started for install instructions.
