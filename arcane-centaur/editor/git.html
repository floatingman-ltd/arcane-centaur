Git Guide

This configuration bundles three complementary git tools that work together as a workflow: vim-fugitive for full git operations, gitsigns.nvim for in-buffer hunk management, and diffview.nvim for side-by-side diffs and file history.

Quick Start

  1. Press <leader>gs to open the fugitive status window.

  2. Navigate to a file and press s to stage it (or u to unstage).

  3. Press cc to open the commit message editor.

  4. Write your commit message, then save and quit (:wq) to commit.

  5. Use <leader>gp to push.

Reviewing Changes

In-buffer signs (gitsigns)

Changed lines are marked in the sign column:

  • ▎ added

  • ▎ changed

  • ▁ / ▔ deleted line

Navigate between hunks with ]h (next) and [h (previous).

Full diff view (diffview.nvim)

Keys Mode Action

<leader>gD

Normal

Open diffview — all unstaged changes

<leader>gX

Normal

Close diffview

The diffview panel shows all changed files on the left; click any file to open a side-by-side diff. Diffview also supports comparing against any git ref:

:DiffviewOpen HEAD~3    " diff last 3 commits
:DiffviewOpen main      " diff against main branch

Staging Hunks

Stage (or reset) individual hunks without opening the fugitive status:

Keys Mode Action

<leader>hs

Normal / Visual

Stage hunk (or selected lines)

<leader>hr

Normal / Visual

Reset hunk (or selected lines)

<leader>hu

Normal

Undo last stage

<leader>hp

Normal

Preview hunk inline

In Visual mode, only the lines in the selection are staged / reset.

Committing with Fugitive

Keys Action <leader>gs

Open git status (:Git)

s (inside status)

Stage file / hunk under cursor

u (inside status)

Unstage file / hunk

= (inside status)

Toggle inline diff for file under cursor

cc (inside status)

Commit staged changes — opens message editor

Normal

Git push

Resolving Conflicts

Use diffview.nvim’s merge-conflict panel:

:DiffviewOpen     " opens showing unresolved conflicts with a 3-way layout

In the merge panel: * Left pane — OURS * Middle pane — working copy (edit here) * Right pane — THEIRS

Stage the file with <leader>hs (or from :Git) once resolved.

File History

Keys Mode Action

<leader>gH

Normal

Open diffview file history for current file

<leader>gX

Normal

Close diffview

The file history panel lists all commits that touched the current file. Press Enter on any commit to open a diff of that commit.

Typical Workflow

  1. Edit code.

  2. <leader>hs — stage individual hunks as you work.

  3. <leader>gs — open status window for a final review.

  4. cc — write commit message and commit.

  5. <leader>gp — push.

  6. <leader>gD — verify the diff before and after.

Setup

Git itself must be installed and on your $PATH:

sudo apt install git
git --version

No additional plugin configuration is required — all three plugins are loaded automatically when Neovim starts.
