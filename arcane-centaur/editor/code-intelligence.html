Using the panel

Every <leader>x* map is a toggle: press it once to open the panel, again to close it. Inside the panel, j/k move between entries, Enter jumps to the location under the cursor (the panel stays open), q closes the panel, and ? lists every in-panel keybinding (preview, fold, filter, delete, …).

Pick the source for the job:

Panel What it shows

Project diagnostics (<leader>xx)

Every LSP warning/error across the whole workspace

Buffer diagnostics (<leader>xX)

Diagnostics for the current file only

Symbols (<leader>xs)

Outline of the current file (functions, classes, …); opens to the side and leaves the cursor in your code

LSP refs/defs (<leader>xr)

References and definitions of the symbol under the cursor, in a right-hand panel

Location list (<leader>xl)

The current window’s location list

Quickfix (<leader>xq)

The quickfix list — fill it first with gr, :grep, :make, or :lua vim.diagnostic.setqflist()

Todo comments (todo-comments.nvim)

todo-comments.nvim highlights annotation keywords in your comments — coloured text plus a sign in the gutter — and can list them across the whole project. A keyword is only recognised when written with a trailing colon (TODO:) inside a comment.

Keyword Aliases (same colour)

TODO:

—

FIX:

FIXME: BUG: FIXIT: ISSUE:

HACK:

—

WARN:

WARNING: XXX:

PERF:

OPTIM: PERFORMANCE: OPTIMIZE:

NOTE:

INFO:

TEST:

TESTING: PASSED: FAILED:

Two list views scan the whole project for these markers:

  • <leader>xt (:TodoTrouble) — todos in a Trouble panel, navigated exactly like the panels above.

  • <leader>xT (:TodoFzfLua) — todos in an fzf-lua fuzzy picker; type to filter, Enter to jump.

Both views search with ripgrep, so rg must be on your PATH; the fzf-lua view additionally needs the fzf binary. If either is missing the map reports it (rg was not found / 'fzf' not installed) rather than listing todos.
[d/]d and <leader>e (LSP diagnostics float) are unchanged. ]t/[t are tag navigation, covered in Tag Navigation (ctags) below.

Configured in lua/plugins/trouble.lua and lua/plugins/todo-comments.lua.

Tag Navigation (ctags)

A tags file is an index — generated by universal-ctags — that maps every symbol name to the file and line where it is defined. It gives cross-file "jump to definition" in any language with no language server involved, and it is what makes the ]t/[t maps do anything.

Generate the index at the project root (Neovim finds ./tags automatically). The index is a static snapshot, so re-run this after significant edits:

ctags -R          # writes ./tags (gitignored)

universal-ctags is a soft dependency — see System Dependencies to install it.

Keys Action

Ctrl-]

Jump to the definition of the tag under the cursor

g Ctrl-]

Same, but list the matches to choose from when there are several (:tjump)

Ctrl-o

Jump back to where you came from (the jump list)

]t / [t

Next / previous match when a tag has several definitions (vim-unimpaired → :tnext / :tprevious)

]T / [T

Last / first match (:tlast / :tfirst)

:tag {name}

Jump to a named tag; :tag /pat jumps to the first tag matching a pattern

:tselect {name}

List all matching tags and pick one
]t/[t only move when the current tag has more than one definition — with a single match there is nothing to cycle to (you may see E428/E425 at the list ends, which is normal). In LSP buffers, gd (go to definition) is usually the better tool and needs no tags file.
Vim’s built-in Ctrl-t (pop the tag stack) is remapped in this configuration to toggle the file tree (:NvimTreeToggle). To step back after a tag jump use Ctrl-o or :pop instead.

`]t`/[t/]T/[T come from tpope/vim-unimpaired; Ctrl-], g Ctrl-], and Ctrl-o are Vim built-ins.

Auto-Completion (blink.cmp)
