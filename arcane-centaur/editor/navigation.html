IDE Layout

Press <leader>L to assemble the IDE workspace in one keystroke:

┌────────────┬───────────────────┐
│  nvim-tree │      editor       │
├────────────┴───────────────────┤
│   terminal (full width, 15)    │
└────────────────────────────────┘

The operation is idempotent — invoking it when the layout is already assembled does nothing, and focus always returns to the editor window.

Layout stability

  • :Bd — layout-preserving buffer delete. Switches the window to another listed buffer (or a new empty one) before deleting, so the window and the tree’s width survive.

  • The tree closes automatically when quitting would leave it (and/or the terminal panel) as the only remaining window — no more full-width tree after :q on the last editor.

Terminal

Keys Mode Action

Esc

Terminal

The terminal always opens as a full-width bottom split regardless of which window has focus. The shell buffer is reused across open/close cycles — history and running jobs survive window toggles. Press <C-k> after Esc to jump back to the editor window.

Treesitter Text Objects (nvim-treesitter-textobjects)

Text objects and motions are available in F#, Haskell, C#, and Lua buffers. They are disabled in Lisp-family buffers (lisp/clojure/scheme/fennel/janet) where vim-sexp provides structural af/if form objects instead.

This config tracks nvim-treesitter’s maintained main branch (not the frozen master), which requires Neovim ≥ 0.11 — already satisfied by this config’s ≥ 0.12 baseline.

Select objects

Use with any operator (d, c, y, v):

Keys Object

af / if

Outer / inner function

ac / ic

Outer / inner class

aa / ia

Outer / inner argument (treesitter’s @parameter capture)

Examples: daf deletes a whole function; vif selects the function body; cia changes an argument.

Move motions

Jump between functions; each jump is added to the jumplist (Ctrl-o / Ctrl-i to navigate).

Keys Action

]f

Next function start

[f

Previous function start

]F

Next function end

[F

Previous function end
Class motions (]c / [c) are intentionally absent — those keys are reserved for Vim’s built-in diff-mode change navigation.

Plugin: nvim-treesitter/nvim-treesitter-textobjects, pinned to its maintained main branch (queries Neovim’s core treesitter APIs, so it works on 0.12 — the frozen master branch does not). Configured in lua/plugins/treesitter.lua.

The stepping keys only move through entries — you fill the list first with a command. Common real-world producers:

Command What lands in the quickfix list

gr (LSP references)

Every use of the symbol under the cursor. Walk the call sites with ]q before renaming or deleting it.

:lua vim.diagnostic.setqflist()

All LSP errors/warnings across loaded buffers; step through them with ]q and fix each in turn.

:grep {pat} / :vimgrep /{pat}/ */.lua

Project-wide text search. Review hits with ]q, then apply one edit to every match with :cdo s/old/new/g | update.

:make

Compiler / build errors parsed via errorformat — jump straight to each failing line.
To jump to the next/previous occurrence of the word under the cursor — a different job — use Vim’s built-in * / # and n / N (no list needed). ]d / [d step between LSP diagnostics.
