AsciiDoc Cheatsheet

LocalLeader = , in .adoc buffers (filetype: asciidoctor)

→ Back to main cheatsheet · Full AsciiDoc guide

AsciiDoc / Antora is the source of truth for documentation in this config. All docs pages live under docs/modules/ROOT/pages/. The nav entry point is docs/modules/ROOT/nav.adoc.

Preview — Single Page

Converts the current .adoc buffer to HTML using the asciidoctor/docker-asciidoctor Docker image and opens it in the system browser. Requires Docker and a graphical environment.

Keys Mode Action

,p or ,pp

Normal

Convert .adoc to HTML (Docker asciidoctor) and open in browser

Console mode (SSH / headless): both maps show a notification that browser preview requires a GUI.

The asciidoctor/docker-asciidoctor image is pulled automatically on first use (~200 MB).

Preview — Full Antora Site

Builds the complete Antora site from antora-playbook-local.yml using the antora/antora Docker image and opens build/site/index.html. This resolves cross-page xref: links, includes, and the Antora nav — the closest thing to a production preview.

Keys Mode Action

,pa

Normal

Build full Antora site (Docker) and open in browser

Requires Docker, a graphical environment, and antora-playbook-local.yml in the repo root (separate from antora-playbook.yml which targets the remote Git source; the local playbook reads from the working tree).

Build output goes to build/site/ (gitignored).

Syntax and Folding

Provided by habamax/vim-asciidoctor:

  • Section folding — use standard Neovim fold keys (za, zc, zo, zM, zR).

  • Fenced-code-block highlighting: [source,lua] highlights as Lua, etc. Supported languages: lua, bash, python, clojure, fsharp, haskell.

  • Markup conceal is off — raw markup stays visible while editing.

Shell Build

To build the Antora site from the shell (requires Docker):

docker compose -f antora-playbook.yml run --rm antora antora-playbook.yml

LSP / Treesitter

No AsciiDoc LSP is configured. No treesitter parser is included (the vim-asciidoctor syntax plugin is the highlighting source). See Architecture for notes.

Keymaps defined in after/ftplugin/asciidoctor.lua. Plugin configured in lua/plugins/asciidoc.lua.
