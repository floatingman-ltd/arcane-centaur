04 — Code Assistance Workflow

This lesson covers the in-editor workflow: selecting code, sending it to Avante, and getting useful responses for explanation, review, and refactoring — all without leaving Neovim.

Open a file from this config in one split and Avante in another with <leader>ao. A good file to start with: ~/.config/nvim/lua/plugins/avante.lua.

1. The Selection Workflow

The core pattern is: select code → open Avante → ask about the selection.

Avante picks up visually selected text automatically when you invoke it while in visual mode.

Step Action

1. Select code

Enter visual mode (v, V, or <C-v>) and select the lines you want

2. Open Avante

Press <leader>aa (or <leader>ao to also switch to ollama)

3. Ask

Type your question in the input area — the selection is included as context

Try it:

  1. Open lua/plugins/avante.lua

  2. Select the entire keys = { …​ } block with V + motion

  3. Press <leader>ao

  4. Ask: Explain what each of these keymaps does.

Confirm you see: an explanation of <leader>aa, <leader>ao, and <leader>ac and their Lua implementations.

2. Explaining Code

Use Avante to explain unfamiliar code. Good candidates in this config:

lua/plugins/ufo.lua — the fold provider selector:

  1. Open lua/plugins/ufo.lua

  2. Select the provider_selector function body

  3. Press <leader>ao

  4. Ask: Why does markdown get different fold providers than other filetypes?

Confirm you see: an explanation of why { "treesitter", "indent" } is used for markdown (marksman provides no fold ranges, no markdown fold queries in treesitter).

lua/config/lsp.lua — the native LSP API:

  1. Open lua/config/lsp.lua

  2. Select the vim.lsp.config block for any server

  3. Ask: What is the difference between vim.lsp.config and lspconfig.setup?

3. Reviewing Code

Avante can act as a code reviewer. The prompting techniques from lesson 03 apply directly.

Pattern: paste or select, then ask for a review with a specific focus

  1. Open lua/options.lua

  2. Select the clipboard detection block (lines with vim.g.clipboard and OSC 52)

  3. Press <leader>ao

  4. Ask: Review this for correctness and edge cases. Are there any scenarios where this could silently fail?

Confirm you see: a review focusing on potential failure modes rather than style.

For a security-focused review, set a role first (new Avante session):

You are a security-focused Neovim plugin reviewer.

Then select and ask:

Review this Docker Compose config for any security concerns.

(Open docker/ollama/docker-compose.yml and select the whole file.)

4. Suggesting Refactors

Ask Avante to suggest improvements to specific code.

  1. Open lua/config/lsp.lua

  2. Select the full file with ggVG

  3. Press <leader>ao

  4. Ask: Are there any improvements you’d suggest to make this more idiomatic for Neovim 0.11+?

Confirm you see: suggestions (or confirmation that the native API is already idiomatic).

Important: treat refactor suggestions as input, not instructions. Review what Avante proposes before applying anything. It may suggest changes that conflict with intentional design decisions in this config.

5. Getting Completions for New Code

You can ask Avante to write new code in the style of the existing config.

  1. Open lua/config/lsp.lua

  2. Select the vim.lsp.config("lua_ls", …​) block

  3. Press <leader>ao

  4. Ask: Using this exact pattern, write an equivalent block for the `zls Zig language server. Use port 0 and no extra settings.`

Confirm you see: a vim.lsp.config("zls", …​) block following the same structure.

6. Tips for Effective Code Assistance

  • Be specific about what you want — "explain" vs "review" vs "refactor" produce very different responses

  • Select the minimum context — don’t select the whole file if one function is enough

  • Ask about one thing at a time — compound questions ("explain AND refactor AND add tests") produce scattered answers

  • Ollama 8B is strong for explanation; weaker for complex generation — switch to <leader>ac (copilot) for harder generation tasks

  • Always review generated code — the model does not know your full codebase
