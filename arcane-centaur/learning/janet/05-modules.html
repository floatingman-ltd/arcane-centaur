05 — Modules with Janet

Janet’s module system lets you split code across files, share libraries via jpm, and control what gets exported. This lesson covers import, use, creating your own modules, and the standard jpm project layout.

Open a scratch file and the Conjure log before starting:

nvim scratch.janet
,lv

1. What Is a Module?

In Janet, any .janet file is a module. When you import a file, Janet executes it and returns its environment as a table of bindings.

# mathlib.janet
(defn square [x] (* x x))
(defn cube   [x] (* x x x))
# main.janet
(import ./mathlib)
(mathlib/square 5)   # => 25
(mathlib/cube 3)     # => 27

The module name becomes a prefix for all its exported symbols.

2. import

import loads a module and binds its public symbols under a namespace prefix.

# Import with default prefix (module file name)
(import spork/fmt)
(spork/fmt/format-print "(def x 1)")

# Import with a custom prefix using :as
(import spork/fmt :as fmt)
(fmt/format-print "(def x 1)")

# Import only specific symbols
(import spork/fmt :only [format-print])
(format-print "(def x 1)")

# Import and expose without a prefix (like use, but explicit)
(import spork/path :prefix "")
(join "/home/user" ".config")   # => "/home/user/.config"

Try it: install spork with jpm install spork, then evaluate (import spork/path :as path) followed by (path/join "/a" "b") with ,ee.

3. use

use imports all public symbols from a module directly into the current scope, with no prefix. Useful for DSLs and test helpers where prefixes add noise.

(use spork/path)
(join "/home" "user" ".config")   # => "/home/user/.config"
(basename "/home/user/file.janet")   # => "file.janet"

Use use sparingly in library code — it can cause name collisions. Prefer import :as for explicit namespacing.

4. Controlling Exports

By default, every def and defn at the top level of a file is exported. Prefix a binding with _ to make it private (unexported):

# utils.janet

(def- PI 3.14159)         # def- is private (not exported)
(defn- to-radians [deg]   # defn- is private
  (* deg (/ PI 180)))

(defn circle-area [r]     # exported — no leading -
  (* PI r r))
# main.janet
(import ./utils)
(utils/circle-area 5)   # => 78.539...
# (utils/to-radians 90) would error — private

5. jpm Project Layout

jpm is Janet’s package manager. A typical project has this structure:

my-project/
  project.janet      # project metadata and dependencies
  src/
    init.janet       # main module (entry point)
  test/
    test-main.janet  # tests (use testament or judge)

A minimal project.janet:

(declare-project
  :name "my-project"
  :version "0.1.0"
  :dependencies ["https://github.com/janet-lang/spork"])

(declare-source
  :source ["src/init.janet"])

Common jpm commands:

jpm install          # install dependencies
jpm test             # run tests in test/
jpm build            # build native extensions
jpm install spork    # install a specific package

6. The spork Library

spork is the de-facto Janet standard library extension — a collection of utilities not in the core. Install it once:

jpm install spork

Useful modules:

Module What it provides

spork/fmt

Code formatter — pretty-prints Janet source

spork/path

Cross-platform file path manipulation (join, basename, dirname, ext)

spork/test

Minimal test helpers (is, assert-error)

spork/misc

Miscellaneous utilities (clamp, number-to-bytes, bytes-to-number)

spork/regex

POSIX-compatible regular expressions

 
(import spork/path :as path)

(path/join "docs" "learning" "janet")
# => "docs/learning/janet"

(path/basename "/home/user/file.janet")
# => "file.janet"

(path/ext "/home/user/file.janet")
# => ".janet"

Try it: evaluate each path/ call with ,ee.

7. Mini-Project: A Small Module

Create a two-file project in a temp directory and load it from a scratch file.

# /tmp/greetlib.janet

(defn- make-greeting [template name]
  (string/format template name))

(defn hello [name]
  (make-greeting "Hello, %s!" name))

(defn farewell [name]
  (make-greeting "Goodbye, %s." name))
# scratch.janet
(import /tmp/greetlib :as greet)

(greet/hello "Alice")      # => "Hello, Alice!"
(greet/farewell "Bob")     # => "Goodbye, Bob."

Workflow:

  1. Create /tmp/greetlib.janet with the first block.

  2. In scratch.janet, evaluate (import /tmp/greetlib :as greet) with ,er.

  3. Evaluate (greet/hello "Alice") with ,ee.

  4. Add a new function to greetlib.janet, re-import, and call it.
